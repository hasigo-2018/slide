# 教材品質判断基準

この文書は、「AI時代のビジュアルアーキテクト育成教科書」を生成・レビュー・改善する際の品質判断基準を定める。一般的な教材品質だけでなく、ユーザーが実際の確認・修正で重視した判断傾向を、次回以降の生成へ反映するために使う。

## 1. 最終ゴール

受講生が用語や作業手順を覚えるだけでなく、実例を見て問題を発見し、選択肢を比較し、根拠を説明し、自分で最終判断できる状態を目指す。

各テーマは、次の流れにつながっているか確認する。

`概念を知る → 見覚えのある例で理解する → 実物で発見する → AIと比較する → 自分で判断する → 根拠を説明する`

## 2. ユーザーの判断傾向

### 初学者が「見たことがある」と思える例を優先する

- 理論上は正しくても、広告や実務では見かけない不自然なコピーを避ける。
- 奇抜さや気の利いた表現より、ECサイト、サービスサイト、バナー、フォームなどで多くの人が見たことのあるベタな例を優先する。
- 抽象語だけで説明せず、送料無料、クーポン、無料体験、料金、レビュー、申込期限など、状況をすぐ想像できる題材へ置き換える。
- コピーの判断に幅がある場合は、5案程度を提示し、推奨案と理由を示してから確定する。

### 受講生の現在地を否定せず、成長先を示す

- 「できない人」「作れるだけの人」のように、現在の能力を欠点として固定する表現を避ける。
- 比較では、すでに持っている力と、次に加える判断力・説明力を分けて示す。
- 未熟さを責める構成ではなく、「現在地 → 次に身につける力」という成長段階として見せる。
- AI時代の役割変化は、既存の制作スキルを否定せず、その上に設計・評価・方向づけを加える説明にする。

### 省略しすぎず、使えるスペースには理解に必要な文を入れる

- 短くすること自体を目的にしない。空きが十分にある場合は、見出しだけでなく意味が完結する説明文まで入れる。
- 例文を途中で切らず、実際に使われる自然なコピーとして成立させる。
- ただし、文章を増やす場合も1スライド1メッセージを守り、読み順と比較関係を崩さない。

### 毎年の更新を前提にしない表現を選ぶ

- 教材の本質に不要な「制作から3年」「現在」「今年」など、時間経過で古くなる表現を避ける。
- 継続性を伝える場合は、「継続して活用されている」「長期運用されている」など、経過年数を毎年更新しなくてよい表現を使う。
- 年数や日付が学習上必要な場合は、基準日または確認日を明記し、更新対象であることが分かるようにする。
- 成果情報は、数値データ、担当者ヒアリング、制作者の仮説など、根拠の種類を区別して記載する。

### 実物と分析根拠の一致を重視する

- 実在LPや既存画像を分析する場合、画像内で実際に確認できるコピー、数字、実績、構造だけを根拠にする。
- 画像に写っていない要素を「この画像の分析結果」として書かない。
- 実在画面の正確な観察が学習目的なら、抽象SVGや疑似UIより既存画像・スクリーンショットを優先する。
- 既存アセットを採用する前に内容を確認し、説明したい学習ポイントが実際に見える素材を選ぶ。
- 実例比較では、成果や良し悪しを先に予想させる前に、「誰に」「何を」「どの順番で」「何が見えるか」など、画像から確認できる事実を観察させる。

### 図と文章を補完関係にする

- 図は文章の飾りではなく、文章だけでは想像しづらい構造、比較、実物、変化を見せるために使う。
- 図だけで判断理由が伝わらない場合は、短い判断ポイントを添える。
- 文字が大半を占める場合は、実例画像、Before/After、比較表、工程図へ置き換えられないか検討する。
- 実物がある場合に、説明用の疑似画面を一からSVGで作らない。

### Before / Afterは「同じ条件の見せ方」を比較する

- BeforeとAfterで、商品、条件、価格、期限など比較の前提をそろえる。
- Beforeは得られるメリット、Afterは失う可能性や期限後の状態を、自然な実務コピーで示す。
- メイン例だけでは応用範囲が狭く見える場合、異なる業種・場面の補足例を2件程度追加する。
- ボタン、閉じる操作、選択肢などのUIはコピーの問いかけと一致させる。
- 初学者向けに概念を二分して比較する場合、実際には重なる領域や例外があるなら「学習上の整理」であることを注記し、絶対的な定義として見せない。

### 効果と倫理をセットで扱う

- 行動を促す表現は、効果があるかだけでなく、真実性・可逆性・ユーザー利益を確認する。
- 期限、送料、在庫、利用者数、レビュー、実績、料金条件は、事実に基づく場合のみ使用する。
- 強い損失訴求、偽の希少性、コンファームシェイミングなどは、推奨例として無条件に正解扱いしない。
- クイズで強い訴求を正解にする場合も、解説で倫理的な条件と注意点を必ず示す。

### 各スライドの役割を重複させない

- 連続する説明スライドは、定義、仕組み、応用、実例、判断基準など役割を分ける。
- 同じ結論を言い換えるだけのスライドを続けない。
- 見出しと本文の内容、理論編と実践編、現在テーマと次Blockの順序を一致させる。
- メインコンテンツ以外の強調枠が3つ以上並ぶ場合は、文脈説明を通常本文へ戻し、枠は「最重要ポイント」「注意」「次への接続」など役割が明確なものだけに絞る。

## 3. 生成前に確認する質問

- 初学者が見た瞬間に状況を想像できる題材か。
- 実務で実際に使われる自然な文言か。
- 例は理論を説明するための作り物になりすぎていないか。
- 既存画像や実物画面を使った方が正確で分かりやすくならないか。
- 画像内で見える情報と説明文が一致しているか。
- 文章を短くしすぎて意味が欠けていないか。
- 空いているスペースを、補足例や判断条件に使う価値があるか。
- Before / Afterの前提条件がそろっているか。
- 行動心理学の効果だけでなく、倫理的な使用条件も示しているか。
- 受講生が最後に「自分ならどう判断するか」を説明できる構成か。

## 4. 避けるパターン

- 理論をそのまま文章化した、実務では見かけないコピー。
- 画像に存在しない要素を根拠にした分析。
- 実物画像があるのに、精度の低い疑似UIや抽象SVGで置き換えること。
- 余白があるのに、例文を不自然な一文だけで終わらせること。
- 情報を追加するだけで読み順や左右の対応を崩すこと。
- 効果の強さだけを評価し、誤認・圧力・偽情報の危険を説明しないこと。
- AIの提案を正解として一括反映し、ユーザーの判断を飛ばすこと。

## 5. ユーザー確認の出し方

判断が必要な場合は、次の順で提示する。

1. 現在の問題点
2. 代替案（コピーなど幅がある場合は原則5案）
3. 推奨案
4. 推奨理由
5. 学習効果・レイアウト・倫理上の注意

ユーザーが案を選んだ後、関連するHTML、構成指示書、クイズ、レビュー記録を同時に更新する。

## 6. この基準の更新

単発の修正をすべて恒久ルールにしない。次のいずれかに当てはまる場合に更新する。

- 同じ判断傾向が複数テーマで繰り返された。
- ユーザーが「次回以降も使う」と明示した。
- 学習目的、技術的正確性、倫理性に関わる再発防止ルールである。

更新時は、特定テーマだけの固有例と、全テーマへ適用する判断基準を分けて記載する。
