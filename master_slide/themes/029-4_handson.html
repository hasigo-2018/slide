演習課題：LPデザインの仕様書自動生成

目標

この演習で習得するスキル：Figmaプラグインを使った仕様書の自動生成と、生成前の「ファイル整理」の重要性の体感

課題内容

架空LP「GreenStep」のFigmaファイルを整理し、Design Docプラグインで3レイヤー（スタイルガイド・コンポーネント・レイアウト）の仕様書を自動生成します

制約条件

  • 題材：GreenStep LP（1920×1080px / ヒーロー・特徴・料金・CTA）
  • 理論との連携：3レイヤー構造・GIGO原則を意識
  • 制作時間：20分（整理10分 + 生成10分）
  • 必須出力：カラーパレット / タイポグラフィ一覧 / ボタン仕様
⏱️ タイマーをセットしてスタート！ まず整理から始めること

手順 1 / 3：Figmaファイルの整理（GIGO対策）

操作手順

1. レイヤーパネルを開き、すべてのフレーム名を意味ある名前に変更（例：「Frame 12」→「Hero Section」）

2. 繰り返し使われている要素（ボタン・カード等）を選択し、右クリック →「Create Component」でMain Componentに登録

3. 使用している色を選択 → 右パネルの「Fill」横の「＋」→「Create style」でLocal Styleに登録（名前例：Primary / #2E7D32）

理論編との関連

理論編で学んだ「Garbage In, Garbage Out」の原則そのものです。この整理作業が、生成される仕様書の品質を直接決定します。命名・コンポーネント化・スタイル登録の3つが揃って初めて、AIは正確な仕様書を出力できます。

⚠️ よくある間違い：「整理は後でいいからまず生成してみよう」→ 乱雑なファイルから生成すると、意味不明なドキュメントになり二度手間になる
✅ 正解：生成の前に必ず10分間の整理時間を確保する

手順 2 / 3：Design Docプラグインの実行

操作手順

1. Figmaメニュー → Plugins → Design Doc を起動

2. 「Generate Style Guide」を選択 → 対象フレームを指定 → 「Generate」をクリック

3. 生成されたドキュメントページを確認。カラーパレット・タイポグラフィ・スペーシングが3レイヤーのLayer 1（スタイルガイド）として出力されていることを確認

理論編との関連

理論編の「自動生成の3ステップ」——スキャン → 分類・構造化 → ドキュメント生成がここで実行されます。プラグインがFigmaのデータ構造を読み取り、カテゴリ別に整理して出力する過程を体感してください。

⚠️ よくある間違い：生成結果をそのまま「完成」とする → 理論編で学んだ通り、AIの出力は「事実の記録」のみ。次のステップで「意図の補足」が必要
✅ 正解：自動生成はあくまでドラフト。ここからが人間の仕事

手順 3 / 3：生成結果の確認と不足の特定

操作手順

1. 生成されたスタイルガイドページを開き、カラー・フォント・余白の値が正しいか実データと突き合わせる

2. コンポーネント一覧を確認。ボタンの状態（Default/Hover/Disabled）がすべて記録されているかチェック

3. 不足している情報をメモする：「なぜPrimaryカラーを緑にしたのか」「CTAボタンを1画面1つに制限する理由」など

理論編との関連

理論編のレビュー4観点を実践します。正確性（数値の一致）→ 完全性（状態の網羅）→ 意図の補足（設計理由）→ 可読性（読み手への配慮）の順にチェックしてください。

⚠️ よくある間違い：「数値が出ているからOK」→ 数値は合っていても、状態バリエーションの漏れや設計意図の欠落は仕様書の致命的な欠陥
✅ 正解：4観点のチェックリストに沿って確認する

演習課題：仕様書のレビューと意図の補足

目標

この演習で習得するスキル：AIを活用した仕様書レビューと、設計意図を的確に言語化する力

課題内容

演習1で生成した仕様書をGeminiに読み込ませ、レビュー4観点（正確性・完全性・意図・可読性）で確認。不足していた設計意図をGeminiの支援で言語化します

制約条件

  • 入力：演習1で生成した仕様書の内容（テキストとしてコピペ）
  • 理論との連携：レビュー4観点・AIと人間の役割分担
  • 制作時間：15分
  • 必須出力：レビュー結果 / 設計意図3件以上の言語化
⏱️ タイマーをセットしてスタート！

手順 1 / 3：仕様書レビュープロンプト

Geminiに入力するプロンプト例

以下はFigmaプラグインで自動生成したLP「GreenStep」のデザイン仕様書です。 4つの観点でレビューしてください： 1. 正確性：数値やカラーコードに不整合はないか 2. 完全性：ボタンの状態（Hover/Active/Disabled）やレスポンシブ設定に漏れはないか 3. 意図の不足：「なぜそうしたか」の説明が欠けている箇所はどこか 4. 可読性：エンジニアが読んで分かりにくい箇所はないか 仕様書内容： [ここに演習1で生成した仕様書のテキストを貼り付け]

理論編との関連

理論編のレビュー4観点をそのままプロンプトの構造に落とし込んでいます。AIに具体的な観点を指定することで、汎用的なレビューではなく仕様書に特化したチェックが得られます。

⚠️ よくある間違い：「この仕様書をレビューして」だけでは抽象的な回答しか返らない
✅ 正解：4つの観点を明示し、仕様書テキストを丸ごと貼り付ける

手順 2 / 3：設計意図の言語化プロンプト

Geminiに入力するプロンプト例

LP「GreenStep」のデザイン判断について、エンジニア向けの設計意図を言語化してください。 以下の各項目について「なぜそうしたのか」の理由を、認知科学やUXの観点を交えて3〜4文で説明してください： 1. Primaryカラーを緑（#2E7D32）にした理由 - ターゲット：環境意識の高い20-30代 - ブランドコンセプト：サステナブルな日常 2. CTAボタンを1セクション1つに制限した理由 - ページ構成：ヒーロー→特徴→料金→CTA 3. 本文フォントサイズを16pxにした理由 - デバイス想定：PC中心（モバイルは18px） 各項目の理由を「設計意図メモ」として使える形式で出力してください。

理論編との関連

理論編で学んだ「AIにできないこと = 意図の説明」を、ここで人間がGeminiの支援を受けながら補完します。ただしGeminiの出力をそのまま使うのではなく、自分の判断で編集・加筆することが重要です。

⚠️ よくある間違い：Geminiの出力をコピペして「設計意図完成」→ AIは一般論を書くが、このプロジェクト固有の判断理由は人間にしか書けない
✅ 正解：Geminiの出力を「たたき台」として使い、自分の言葉で加筆修正する

手順 3 / 3：引継ぎ文の確認と調整

操作手順

1. Geminiから返ってきたレビュー結果を読み、指摘された漏れ（特に状態バリエーションやレスポンシブ設定）を確認する

2. 設計意図の言語化結果を読み、「自分のプロジェクト固有の理由」が反映されているか確認。一般論すぎる部分を具体的な文言に書き換える

3. レビュー結果 + 設計意図を統合し、「仕様書に追記すべき内容リスト」として整理する

理論編との関連

理論編の協業ワークフロー STEP 3「レビュー・補足」の実践です。AIが生成した事実の記録に、人間が意図と判断を加えることで、初めて実用的な仕様書になります。

⚠️ よくある間違い：レビュー結果だけ見て満足し、仕様書本体に反映しない
✅ 正解：レビュー結果は「追記すべき内容リスト」に変換し、仕様書に実際に書き込む

演習課題：開発引継ぎサマリーの作成

目標

この演習で習得するスキル：仕様書の情報を「伝わる資料」に変換する構成力と、デザイン4原則を自分の資料に適用する力

課題内容

演習1・2で作成した仕様書情報（スタイルガイド + コンポーネント仕様 + 設計意図）を、Canvaで「開発チーム向け引継ぎ1枚サマリー」として仕上げます

制約条件

  • サイズ：1920 × 1080px（プレゼンテーション形式）
  • 使用する原則：整列（グリッド配置）/ 近接（情報グループ化）/ 反復（統一フォント・色）/ コントラスト（セクション見出し）
  • 制作時間：15分
  • 必須要素：カラーパレット / タイポグラフィ一覧 / 主要コンポーネント仕様 / 設計意図メモ
⏱️ タイマーをセットしてスタート！

手順 1 / 3：テンプレート選択と4分割レイアウト

操作手順

1. Canvaで「プレゼンテーション」→「空のプレゼンテーション（16:9）」を選択

2. ページを4つのエリアに分割する構成を決める：左上「カラーパレット」/ 右上「タイポグラフィ」/ 左下「主要コンポーネント」/ 右下「設計意図メモ」

3. ガイドラインを引く：水平の中央線と垂直の中央線を引き、4分割の基準を作る

デザイン原則との関連

4分割レイアウトは「整列」と「近接」を同時に実現します。グリッドで要素を揃え（整列）、関連情報をエリアごとにまとめる（近接）ことで、一目で全体像が把握できます。

⚠️ よくある間違い：情報を思いつくまま並べてしまう → 必ず先に「4エリアの分割」を決めてから情報を入れる
✅ 正解：構造（箱）を先に作り、そこに情報を流し込む

手順 2 / 3：仕様情報の配置と視覚化

操作手順

1. 左上エリア：演習1のカラーパレットをCanvaの「図形」で再現。正方形を並べてHEXコード + 用途ラベル（Primary / Secondary / Accent）を記入

2. 右上エリア：タイポグラフィ一覧をテキストボックスで作成。H1〜H4・Body・Captionの各サイズとフォント名を実際のスタイルで表示

3. 左下エリア：主要コンポーネント（ボタン3状態等）を表形式で記載。Canvaの「表」機能を使用

デザイン原則との関連

「反復」と「コントラスト」の適用です。各エリアで同じフォント・余白を使い（反復）、セクション見出しだけ色と太さを変える（コントラスト）ことで、プロフェッショナルな統一感が出ます。

⚠️ よくある間違い：Canvaの装飾機能（グラデーション、アイコン大量配置等）に走る → 仕様書の引継ぎ資料では「正確さと読みやすさ」が最優先
✅ 正解：装飾は最小限。情報の明確さに全力を注ぐ

手順 3 / 3：設計意図メモの配置と最終確認

操作手順

1. 右下エリア：演習2でGeminiの支援を受けて言語化した設計意図メモ（3件以上）を箇条書きで配置

2. 全体を俯瞰し、4つのエリアの情報量バランスを確認。偏りがある場合はフォントサイズや余白で調整

3. タイトルを追加：「GreenStep LP デザイン仕様サマリー」+ 日付 + 作成者名を上部に配置

理論編との関連

理論編の協業ワークフロー STEP 4「チーム共有」の実践です。読み手（エンジニア）が「何を見ればいいか」が一目で分かる資料になっているかが最終チェックポイントです。

⚠️ よくある間違い：「設計意図」エリアを空白のままにする → 理論編で学んだ通り、AIが書けない「なぜ」こそが仕様書の最も重要な部分
✅ 正解：設計意図メモは必ず記載。これが引継ぎ資料の価値を決める
