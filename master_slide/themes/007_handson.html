Tool Setup
ClarityUX™ 導入

Figmaプラグインのインストールと基本操作

ClarityUX™ とは

用途：AIによる視線予測・デザインレビュー・A/Bテスト

特徴：Figma内で完結する多機能プラグイン

主要機能

  • Attention Heatmap：ユーザーがどこを見るか予測
  • Clarity Score：読みやすさ・UXをスコア化
  • A/B Testing：デザイン比較を数値で判断
  • WCAG Check：アクセシビリティ検証
💡 無料トライアルで今日の演習は十分対応可能

手順 1：プラグインのインストール

操作手順

1. Figmaを開き、任意のデザインファイルを開く

2. 上部メニュー「Resources」→「Plugins」→ 検索窓に「ClarityUX」と入力

3. 「ClarityUX™: AI Design Review...」を選択 →「Run」をクリック

4. 初回起動時にアカウント登録（メールアドレスのみ）

代替アクセス方法

Exercise演習：LP改善サイクル

サービスLPのCTAを視線予測で検証・改善する

演習課題：LPのCTA検証と改善

シナリオ

あなたはサービスLPを制作しました。CTAボタンを配置しましたが、
「本当にユーザーの目に入るか？」を検証せずに公開しようとしています。

課題内容

視線予測AIを使って、CTAの視認性を検証し、問題があれば改善する

検証ポイント（理論編の復習）

  • CTA配置：視線の終着点にあるか？
  • CTAコントラスト：周囲と差別化されているか？
  • 情報の優先順位：重要な情報が先に目に入るか？
⏱️ 所要時間：約20分（分析5分 + 改善10分 + 再分析5分）

手順 2：分析対象のフレームを準備

選択肢A：自分のデザインを使う

制作中のLP・Webページがあれば、そのフレームを選択

選択肢B：サンプルを作成する

以下の要素を含むシンプルなLPフレームを作成（5分）：

  • ヘッドライン（サービス名 + キャッチコピー）
  • 3つの特徴（アイコン + テキスト）
  • CTAボタン（「無料で始める」など）
  • フレームサイズ：1440 × 900px（Desktop）

操作手順

1. 分析したいフレームを選択（クリック）

2. フレーム全体が選択されていることを確認（レイヤーパネルで確認）

⚠️ よくある間違い：グループやコンポーネントを選択している
✅ 必ず最上位の「Frame」を選択

手順 3：ヒートマップを生成する

操作手順

1. フレームを選択した状態で右クリック

2. 「Plugins」→「ClarityUX」→「Run」

3. プラグインパネルで「Heatmap」または「Analyze」を選択

4. 数秒待つと、ヒートマップが生成される

生成されるデータ

  • Attention Heatmap：赤（高注目）→ 黄 → 緑 → 青（低注目）
  • Clarity Score：デザインの読みやすさスコア（0〜100）
  • Focus Areas：注目を集めている要素のリスト
💡 ヒートマップはFigmaのレイヤーとして追加される
→ 表示/非表示を切り替えて比較可能

手順 4：ヒートマップの読み方

手順 5：問題点を特定する

よくある問題パターンと原因

問題1：CTAが目立たない（青/緑）

  • 原因A：周囲と同系色でコントラスト不足
  • 原因B：サイズが小さすぎる
  • 原因C：視線の流れから外れた位置にある

問題2：関係ない要素が目立つ（赤）

  • 原因A：装飾画像が派手すぎる
  • 原因B：背景パターンが視線を奪っている

問題3：Clarity Scoreが低い（70未満）

  • 原因A：情報が散らばっている（近接の法則違反）
  • 原因B：視覚的階層が不明確
📝 問題を1つ選び、理論編の原則に基づいて改善案を考える

手順 6：改善を実施する

改善アプローチ（理論編の原則を適用）

CTAが目立たない場合：

  • コントラスト強化：ページ基調色の補色に変更
  • サイズ拡大：横幅200px以上、高さ50px以上に
  • 位置調整：視線の終着点（Z型ならページ下部右側）へ移動
  • 余白追加：CTAの周囲に十分な空間を確保

関係ない要素が目立つ場合：

  • 彩度を下げる / グレースケールに近づける
  • サイズを縮小する
  • 背景に溶け込ませる

操作手順

1. 元のフレームを複製（Ctrl/Cmd + D）→「After」と命名

2. 複製したフレームで改善を実施

3. Before/Afterを並べて配置

⚠️ 一度に複数の変更をしない → 何が効果的だったか分からなくなる

手順 7：再分析してBefore/Afterを比較

操作手順

1. 改善後のフレーム（After）を選択

2. 再度ClarityUXでヒートマップを生成

3. Before/Afterのヒートマップを並べて比較

確認ポイント

  • CTAの注目度は上がったか？（青/緑 → 黄/赤）
  • Clarity Scoreは改善したか？
  • 意図しない副作用はないか？（他の重要要素が埋もれていないか）

A/B Testing機能の活用

ClarityUXの「A/B Testing」機能を使うと、2つのフレームを
自動で比較し、どちらが優れているかスコアで判定できます。

💡 改善が見られない場合 → 別の原則を適用して再挑戦
