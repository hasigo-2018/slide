AI×開発連携ドキュメント【実践編①】

Figma・Gemini・Claudeで仕様書を自動生成する

本日のゴール

共通題材：ECサイトの「カート追加ボタン」

1つのUIコンポーネントを題材に、3種類の仕様書をAIで生成します。

3つの演習

  • 演習1：UIスペック生成（Figma + Gemini）— 15分
  • 演習2：コンポーネント仕様書生成（Claude）— 15分
  • 演習3：ハンドオフ資料作成（Figma + Claude）— 20分
🎯 最終成果物：GitHub Issueに投稿できるハンドオフ資料

演習1
UIスペック生成

Figma + Gemini で数値を正確に記録する

演習課題：UIスペックシート作成

目標

Figmaからデザイン情報を抽出し、Geminiで構造化されたUIスペックを生成する

課題内容

「カート追加ボタン」のUIスペックをMarkdownテーブル形式で作成します

制約条件

  • 対象：ボタン1つ（カート追加ボタン）
  • 必須項目：カラー、タイポグラフィ、サイズ、スペーシング
  • 出力形式：Markdownテーブル（GitHub対応）
  • 制作時間：15分
⏱️ タイマーをセットしてスタート！

手順 1 / 3：Figmaでデザイン情報を取得

操作手順

1. Figmaでボタンコンポーネントを選択

2. 右パネルの「Design」タブで以下を確認：

  • Fill（背景色）のHEXコード
  • Text（文字色）のHEXコード
  • フォント名、サイズ、ウェイト
  • Auto Layout のパディング値
  • 幅・高さ（固定 or Hug）

3. 値をメモ帳にコピーしておく

⚠️ よくある間違い：目視で「だいたい16px」と推測する
✅ 正解：必ずFigmaの数値を確認する

手順 2 / 3：Geminiでスペックを整形

Geminiに入力するプロンプト例

以下のデザイン情報をUIスペックシートに整形してください。 GitHub Issue用のMarkdownテーブル形式で出力してください。 【ボタン情報】 - 背景色：#00d4ff - 文字色：#1a1a2e - フォント：Noto Sans JP - フォントサイズ：14px - フォントウェイト：Bold - パディング：上下12px、左右24px - 角丸：4px - 幅：Hug contents 【出力形式】 | カテゴリ | プロパティ | 値 | の形式でお願いします。
💡 ポイント：出力形式を明示すると、そのまま使える形で返ってくる

手順 3 / 3：生成結果の確認と修正

確認ポイント

1. すべての数値が正確に反映されているか

2. カテゴリ分けが適切か（カラー / タイポグラフィ / サイズ / スペーシング）

3. 抜け漏れがないか（hover状態の色など）

修正が必要な場合

Geminiに追加指示：「hover時の背景色 #00b8e6 も追加してください」

⚠️ よくある間違い：AI出力をそのまま使う
✅ 正解：必ず元のFigmaデータと照合する

演習2
コンポーネント仕様書生成

Claude で動作と状態を定義する

演習課題：コンポーネント仕様書作成

目標

ボタンの状態変化とProps定義をClaudeで生成する

課題内容

「カート追加ボタン」の全状態と振る舞いを定義した仕様書を作成します

制約条件

  • 対象：カート追加ボタン（5つの状態を持つ）
  • 状態：Default / Hover / Active / Disabled / Loading
  • 出力形式：TypeScript型定義 + 状態説明
  • 制作時間：15分
⏱️ タイマーをセットしてスタート！

手順 1 / 3：Claudeにコンポーネント情報を伝える

Claudeに入力するプロンプト例

ECサイトの「カートに追加」ボタンのコンポーネント仕様書を作成してください。 【ボタンの基本情報】 - 用途：商品詳細ページでカートに商品を追加する - テキスト：「カートに追加」 【必要な状態】 1. Default：通常状態 2. Hover：マウスオーバー時 3. Active：クリック中 4. Disabled：在庫切れ時 5. Loading：カート追加処理中 【出力形式】 1. TypeScriptのProps型定義（interface） 2. 各状態の説明（どんな時にその状態になるか） 3. アクセシビリティ要件（aria属性など）
💡 ポイント：「用途」を伝えると、文脈に合った提案が返ってくる

手順 2 / 3：UIスペックを追加して精度を上げる

演習1の結果を活用する

先ほどの仕様に、以下のUIスペックを追加してください。 【UIスペック】 - Default背景：#00d4ff - Hover背景：#00b8e6（10%暗く） - Active背景：#009cc7（20%暗く） - Disabled背景：#4a4a5e、opacity: 0.5 - Loading時：spinnerアイコン表示、テキスト「追加中...」 各状態ごとのスタイル変更を仕様書に追記してください。
⚠️ よくある間違い：一度に全部伝えようとして情報が散らかる
✅ 正解：段階的に情報を追加して精度を上げる

手順 3 / 3：エッジケースを追加

開発者が知りたい「例外処理」

Claudeに追加で質問：

  • 「在庫が残り1点の時、ボタン表示はどうなりますか？」
  • 「カート追加に失敗した時のエラー表示は？」
  • 「すでにカートに入っている商品の場合は？」

確認ポイント

生成された仕様書に以下が含まれているか：

  • Props型定義（TypeScript interface）
  • 5つの状態の説明と遷移条件
  • アクセシビリティ要件
💡 エッジケースを先に定義しておくと、実装後の手戻りが激減する

演習3
ハンドオフ資料作成

Figma + Claude で GitHub Issue を作成する

演習課題：GitHub Issue 形式のハンドオフ資料

目標

演習1・2の成果物を統合し、開発者がすぐに着手できるIssueを作成する

課題内容

「カート追加ボタン実装」のGitHub Issueテンプレートを完成させます

必須セクション

  • ## 実装対象：何を作るか
  • ## Figma：デザインリンク
  • ## UIスペック：演習1の成果
  • ## コンポーネント仕様：演習2の成果
  • ## 実装メモ：技術的な注意点
⏱️ 制作時間：20分

手順 1 / 3：Issue テンプレートを生成

Claudeに入力するプロンプト例

GitHub Issue形式で「カート追加ボタン」の実装タスクを作成してください。 【含めるセクション】 ## 実装対象 ## Figma ## UIスペック ## コンポーネント仕様 ## 実装メモ ## 完了条件 【UIスペック】 （ここに演習1の結果を貼り付け） 【コンポーネント仕様】 （ここに演習2の結果を貼り付け） Markdown形式で、そのままGitHubに貼り付けられる形式で出力してください。
💡 演習1・2の成果物をそのまま貼り付けて統合する

手順 2 / 3：実装メモを追加

Figmaから追加情報を取得

1. Auto Layout の設定（flex方向、gap値）

2. レスポンシブ時の挙動（幅の変化）

3. アニメーション/トランジション（もしあれば）

Claudeに追加で依頼

「このボタンをReact + TailwindCSSで実装する場合の技術メモを追加してください」

⚠️ よくある間違い：デザイン情報だけ渡して実装方針を書かない
✅ 正解：使用技術に合わせた実装ヒントを含める

手順 3 / 3：完了条件を明確化

開発者が「完了」を判断できる条件

Issue の最後に以下を追加：

完了条件の例

  • [ ] 5つの状態（Default/Hover/Active/Disabled/Loading）が実装されている
  • [ ] UIスペック通りの色・サイズになっている
  • [ ] キーボード操作でフォーカスが当たる
  • [ ] スクリーンリーダーで「カートに追加ボタン」と読み上げられる
  • [ ] Storybookにコンポーネントが登録されている
💡 チェックボックス形式にすると、PRレビュー時に確認しやすい
