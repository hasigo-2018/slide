Step 1

Gemini でLP構成を企画する

ツール紹介：Gemini

用途

AIとの対話でLP構成案・キャッチコピー・セクション内容を企画

アクセス方法

URL：https://gemini.google.com/app

Googleアカウントでログインして使用

今日使う機能

  • テキスト生成：LP構成案、キャッチコピーの提案
  • 対話的改善：提案を評価し、修正依頼を繰り返す
  • 制約の指定：ターゲット、トーン、避けたい表現を明示
💡 無料プランで十分使えます

演習課題：SLOW DRIP LPの構成企画

目標

AIを使ってLP全体の構成案とキャッチコピーを決定する

SLOW DRIPの設定

  • 店名：SLOW DRIP（スロードリップ）
  • コンセプト：「急がない時間」を提供するスペシャルティコーヒー専門店
  • 立地：都心のオフィス街から少し離れた住宅地
  • ターゲット：30〜40代のリモートワーカー、クリエイター

制約条件

  • セクション数：5つ（ヒーロー、特徴、メニュー、アクセス、CTA）
  • 制作時間：15分

手順 1：LP構成案を生成

Geminiに入力するプロンプト例

以下のカフェのLP（ランディングページ）構成案を作成してください。 【カフェ情報】 - 店名：SLOW DRIP - コンセプト：「急がない時間」を提供するスペシャルティコーヒー専門店 - ターゲット：30〜40代のリモートワーカー、クリエイター - 立地：都心から少し離れた静かな住宅地 【出力してほしいもの】 1. ヒーローセクションのキャッチコピー（メイン + サブ） 2. 3つの特徴セクションの見出しと説明文（各50字以内） 3. CTAボタンのテキスト案（3パターン） 【避けたいトーン】 - チェーン店のような画一的な表現 - 「最高の」「究極の」などの誇大表現
⚠️ よくある間違い：「LPを作って」だけでは汎用的な回答しか得られない
✅ 正解：ターゲット、トーン、出力形式を具体的に指定する

手順 2：提案を評価・改善

AIの回答を評価し、改善を依頼

ありがとうございます。以下の点を修正してください： 1. キャッチコピーがまだ一般的すぎます。 「急がない」というコンセプトをもっと具体的に表現してください。 例：時間の流れ方、コーヒーを淹れる所作、思考の余白など 2. 特徴セクションの説明文が長いです。 各30字以内に短縮してください。 3. CTAは「予約」ではなく「体験」を促す表現に変えてください。 初めての方が気軽に来店できるニュアンスで。
💡 Visual Architectの役割：AIの提案をそのまま採用せず、ブランドに合うか評価・判断する

Step 2

Figma Design でデザインを作成

ツール紹介：Figma Design

用途

LP のワイヤーフレーム作成、デザイン制作、コンポーネント設計

アクセス方法

URL：https://www.figma.com/

「New design file」から新規ファイルを作成

今日使う機能

  • Frame：ページ全体とセクションの枠組み作成
  • Auto Layout：レスポンシブ対応の自動レイアウト
  • Components：ボタン、カードなど再利用可能なパーツ
💡 Starterプランで3ファイルまで無料

演習課題：SLOW DRIP LPデザイン

目標

Figma Sitesで公開できる状態のデザインを作成する

制約条件

  • フレームサイズ：Desktop 1440px幅、縦は可変
  • セクション数：5つ（Geminiで決めた構成）
  • 必須要素：ヒーロー画像、特徴カード×3、CTAボタン
  • 制作時間：20分

カラー設定（SLOW DRIP ブランド）

  • メイン：#2C3E50（ダークネイビー）
  • アクセント：#D4A574（コーヒーベージュ）
  • 背景：#F8F6F3（オフホワイト）

手順 1：フレーム作成とAuto Layout設定

操作手順

1. Frame tool（F）で「Desktop」プリセットを選択（1440×1024）

2. フレーム高さを「Hug contents」に変更（内容に合わせて伸縮）

3. Auto Layoutを適用（Shift + A）し、Direction を「Vertical」に

4. Paddingを左右80px、上下0pxに設定

なぜAuto Layoutが重要か

Figma Sitesで公開する際、Auto LayoutがそのままCSSのFlexboxに変換されます。
ここでの設定が表示崩れを防ぐ鍵になります。

⚠️ よくある間違い：固定サイズでレイアウトを組む
✅ 正解：Auto Layout + Hug/Fill で柔軟なレイアウトを作る

手順 2：セクションとコンポーネント作成

操作手順

1. 各セクション用のFrame を作成し、Auto Layout を適用

2. ヒーローセクション：高さ600px、背景画像 + オーバーレイ

3. 特徴カード：Auto Layout（横並び）、Gap 32px

4. CTAボタン：Padding 16px 32px、角丸8px でコンポーネント化

デザイン原則の適用

  • 近接：関連する情報（見出し + 説明）を近づける
  • 整列：すべてのセクションを中央揃えで統一
  • 反復：同じスタイルのカードを3つ並べる
⚠️ よくある間違い：Auto Layout内の要素サイズを固定値にする
✅ 正解：Width を「Fill container」にして親に追従させる

手順 3：レスポンシブ準備

操作手順

1. Desktopフレームを複製し、幅を768px（Tablet）に変更

2. 特徴カードのAuto Layout Direction を「Vertical」に変更

3. さらに複製し、幅を375px（Mobile）に変更

4. フォントサイズ、Paddingを調整

Figma Sitesでの動作

Figma Sitesはブレークポイントごとに異なるレイアウトを設定できます。
ここで作った3つのフレームがそのままブレークポイントになります。

💡 「リサイズ」ではなく「再設計」の意識を持つ
→ 単に縮めるのではなく、画面サイズに最適化した配置を考える

Step 3

Figma Sites でサイト化・公開

ツール紹介：Figma Sites

用途

Figmaのデザインをそのまま本物のWebサイトとして公開

アクセス方法

新規作成：File browser →「New」→「Site」を選択

既存から変換：Figma Design のフレームをコピー → Sites にペースト

今日使う機能

  • Breakpoints：Desktop / Tablet / Mobile の表示切替
  • Interactions：ホバー、スクロールエフェクトの追加
  • Preview：公開前のブラウザ表示確認
  • Publish：ワンクリックで xxx.figma.site に公開
💡 有料プラン（Professional〜）で公開機能が使えます

演習課題：SLOW DRIP LP公開

目標

Figma Designで作ったデザインを実際にブラウザで閲覧できる状態にする

制約条件

  • ブレークポイント：3つ（Desktop 1280px / Tablet 768px / Mobile 375px）
  • インタラクション：最低1つ（CTAボタンのホバー効果）
  • 制作時間：15分

確認ポイント

  • Previewで3サイズすべての表示を確認
  • テキストの折り返し、画像のはみ出しがないか
  • ボタンのタップ領域が十分か（Mobile）

手順 1：デザインをSitesに移行

操作手順

1. Figma Designでデザインフレーム全体を選択（Cmd/Ctrl + A）

2. コピー（Cmd/Ctrl + C）

3. Figma Sitesの新規ファイルを開く

4. ペースト（Cmd/Ctrl + V）

5. 「Webpage」としてペーストを選択

移行時の注意

Auto Layoutの設定がそのまま引き継がれます。
Figma Design側でAuto Layoutが正しく設定されていないと、ここで崩れが発生します。

⚠️ よくある間違い：絶対配置の要素をそのままペースト
✅ 正解：必ずAuto Layout内に配置してからペースト

手順 2：ブレークポイント設定

操作手順

1. 左パネルの「Breakpoints」をクリック

2. Desktop（1280px〜）、Tablet（768px〜1279px）、Mobile（〜767px）を確認

3. 各ブレークポイントで表示を切り替え、レイアウトを調整

4. Mobile：特徴カードを縦並びに変更（Direction: Vertical）

ブレークポイントごとの調整例

  • Desktop：特徴カード横並び、大きなヒーロー画像
  • Tablet：特徴カード2列 + 1列、マージン縮小
  • Mobile：すべて縦並び、フォントサイズ縮小
💡 Figma Sitesのブレークポイント設定は、そのままCSSの@media queryに変換されます

手順 3：プレビューと公開

操作手順

1. Shift + Space でインラインプレビューを開く

2. ブラウザ幅をドラッグして、各ブレークポイントでの表示を確認

3. 問題なければ、左パネル「File」→「Publish」をクリック

4. サイトタイトル、説明を入力して「Publish」ボタン

公開後の確認

  • 生成されたURL（xxx.figma.site）にアクセス
  • 実際のブラウザ（Chrome, Safari, Firefox）で表示確認
  • スマートフォン実機でも確認（推奨）
⚠️ よくある間違い：プレビューだけで満足する
✅ 正解：必ず公開後のURLを複数ブラウザで確認

Step 4

Code Layers で表示崩れ修正・アニメーション追加

ツール紹介：Code Layers

用途

Figma Sites内でカスタムインタラクション・アニメーションをAIで生成

アクセス方法

Figma Sites内で要素を選択 →「Make code from design」をクリック

今日使う機能

  • AIプロンプト：自然言語でアニメーションを指示
  • コード編集：React + TypeScript + Tailwind CSSを直接編集
  • CSS微調整：表示崩れの根本原因を修正
💡 Code Layersは Figma Make と同じAIエンジン（Claude Sonnet）を使用

演習課題：表示崩れ修正とアニメーション追加

目標

公開したサイトの問題点を特定し、Code Layersで修正・改善する

よくある表示崩れパターン

  • テキスト折り返し：長い単語が枠からはみ出す
  • 画像比率崩れ：アスペクト比が維持されない
  • 余白の不統一：ブレークポイント間でPaddingが不自然
  • ボタンタップ領域：Mobileでタップしにくい

追加するアニメーション

  • CTAボタンのホバーエフェクト（拡大 + 色変化）
  • スクロール時のフェードイン（Reveal効果）

手順 1：CTAボタンにホバーエフェクト追加

Code Layersに入力するプロンプト例

このボタンに以下のホバーエフェクトを追加してください： 1. ホバー時に少し拡大（scale: 1.05） 2. 背景色を少し明るく（現在の色より10%明るく） 3. 軽い影を追加（box-shadow） 4. アニメーションは0.2秒でスムーズに モバイルではホバー効果は無効にしてください。
⚠️ よくある間違い：「かっこいいアニメーションをつけて」と曖昧に指示
✅ 正解：具体的な数値（scale値、秒数）と条件（モバイル除外）を明示

手順 2：表示崩れの修正

Code Layersに入力するプロンプト例

以下の表示崩れを修正してください： 1. 長い英単語がコンテナからはみ出している → word-break: break-word を適用 2. 画像のアスペクト比が崩れている → object-fit: cover を適用 3. モバイルで左右の余白が不均等 → padding-left と padding-right を 16px に統一 修正後、他の要素に影響がないか確認してください。
💡 Visual Architectの役割：問題を「発見」し、具体的な修正方法を「指示」する
コードを書く必要はないが、CSSプロパティ名を知っていると指示が的確になる
