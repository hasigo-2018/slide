SŌILのWFを"部品"に分解して再構築する

演習①：部品の洗い出し ― Geminiで5層分類

目的

011で作ったSŌILのTOPページWFに含まれる要素を、Atomic Designの5層に分類します。まずGeminiに叩き台を出してもらい、それを人間がレビューする流れです。

Geminiに入力するプロンプト

以下はオーガニックスキンケアブランド「SŌIL」のTOPページワイヤーフレームの構成要素です。これらをAtomic Designの5層（Atoms / Molecules / Organisms / Templates / Pages）に分類してください。 【構成要素】 - Header：ロゴ、ナビゲーション（5項目）、CTAボタン「オンラインショップへ」 - Section_01_MV：メインビジュアル画像枠、キャッチコピー、サブコピー、CTAボタン - Section_02_About：セクション見出し、ブランド説明テキスト、画像枠 - Section_03_Products：セクション見出し、商品カード×3（各カード＝画像枠＋商品名＋価格＋「詳しく見る」リンク） - Section_04_Story：セクション見出し、ストーリーテキスト、画像枠 - Section_05_Instagram：セクション見出し、正方形画像枠×6（3×2グリッド）、フォローリンク - Section_06_CTA：見出し、CTAボタン - Footer：ロゴ、ナビゲーションリンク、SNSアイコン×3、コピーライト 各要素がどの層に該当するか、理由とともに一覧表で出力してください。
💡 ポイント：Geminiの出力はあくまで「叩き台」。次のスライドで人間が判断を加えます。

AI出力のレビューと選定

Gemini出力例（分類結果）

Atoms：

  • CTAボタン
  • ナビ項目（テキストリンク）
  • セクション見出し
  • ロゴ
  • SNSアイコン
  • テキストブロック
  • 画像枠

Molecules：

  • 商品カード（画像＋名前＋価格＋リンク）
  • Instagramグリッドセル

Organisms：

  • Header / Footer / 各Section全体

人間が加える判断

AIは「全部」を分類してくれますが、全部をコンポーネント化する必要はありません。

判断基準：「2回以上使うか？」

  • ✅ CTAボタン → MV、CTA、ヘッダーに3回以上
  • ✅ 商品カード → Section_03に×3枚
  • ✅ セクション見出し → 各セクションに1つ
  • ✅ ヘッダー / フッター → 全ページ共通
  • ⬜ ブランド説明テキスト → 1箇所のみ → 後回し
🏗️ アーキテクトの視点：「使い回し頻度」で優先順位を決めるのがプロの判断。

演習②-準備：Componentsページ作成

なぜ別ページを作るのか

メインコンポーネントの管理場所を1箇所にまとめることで、チームの誰もが「部品はここを見ればわかる」状態を作ります。実務では定番の運用手法です。

操作手順

1. 011のSŌIL WFファイルを開く

2. ページパネル（左上）で「＋」をクリック → 新規ページ追加

3. ページ名を Components に変更

4. WFページはそのまま残す（完成後にインスタンスへ差し替え）

ページ構成のイメージ

ファイル内に2ページ：

  • Page 1「TOP WF」：011で作ったワイヤーフレーム（作業用）
  • Page 2「Components」：メインコンポーネントの置き場所（今回新規追加）
⚠️ よくある間違い：WFページ上で直接コンポーネント化してしまうと、メインとインスタンスが混在して管理しにくくなります。必ずComponentsページにメインを集めましょう。

演習②：Atomsを作る ― ボタン＋バリアント

CTAボタンのコンポーネント化

1. Componentsページに移動

2. グレー矩形（W: 200, H: 48, 角丸: 24）を作成し、中央にテキスト「購入する」を配置

3. 矩形＋テキストを選択 → Auto Layout で組む（padding: 上下12 左右32）

4. 選択した状態で Ctrl+Alt+K → コンポーネント作成

5. 名前を Button/Primary に変更

バリアントの追加（Secondary）

6. 右パネルの「+」（Add variant）をクリック

7. 新しいバリアントの塗りを「なし」、枠線を追加（1px、グレー）

8. プロパティ名を Type、値を Primary / Secondary に設定

その他のAtomsも同様にコンポーネント化

  • Heading/Section ― セクション見出し（バリアント不要）
  • Link/Text ― テキストリンク（バリアント不要）
  • Icon/SNS ― SNSアイコン枠（バリアント不要）
⚠️ よくある間違い：Auto Layoutを組む前にコンポーネント化してしまうと、後からAuto Layoutを追加した際にインスタンスとの整合性が崩れることがあります。Auto Layout → コンポーネント化の順序を守りましょう。

演習③-A：Moleculesを作る ― 商品カード

商品カードの構築手順

1. Componentsページで、以下の要素を縦に並べる：

  • 画像枠（矩形 W: 280, H: 200、グレー塗り）
  • 商品名テキスト（例：「フェイスオイル」）
  • 価格テキスト（例：「¥3,800」）
  • Button/Secondary のインスタンスを配置（アセットパネルからドラッグ）

2. 4つの要素を選択 → Auto Layout（縦方向、gap: 12）

3. Ctrl+Alt+K → 名前を Card/Product に変更

WFのカードをインスタンスに差し替え

4. WFページに移動 → Section_03_Productsの商品カード3枚を削除

5. アセットパネルから Card/Product を3つ配置

6. 各インスタンスでテキストを上書き：

  • カード1：フェイスオイル / ¥3,800
  • カード2：クレンジングバーム / ¥2,900
  • カード3：ハンドクリーム / ¥1,600
💡 構造（レイアウト・間隔・ボタン位置）はメインと同期したまま、テキストと画像だけが異なる。これがインスタンスの上書きの力です。

演習③-B：Moleculesを作る ― Instagramグリッド

同じ原理を違う文脈で使う

商品カードと同じ「コンポーネント化 → インスタンス配置」の流れを、まったく違うUIパーツで体験します。

操作手順

1. Componentsページに正方形のグレーボックス（W: 160, H: 160）を作成

2. Ctrl+Alt+K → 名前を Card/Instagram に変更

3. WFページの Section_05_Instagram に移動

4. Card/Instagram のインスタンスを6つ配置

5. 6つのインスタンスを選択 → Auto Layout → Wrapを有効化（3列×2行のグリッド）

ここで気づいてほしいこと

「商品カード」も「Instagram投稿枠」も、やっていることは同じ：

  • 1つのメインコンポーネントを作る
  • 必要な数だけインスタンスを配置する
  • 個別の違いは上書きで対応する
🏗️ コンポーネント設計は「パターン認識」。形が違っても、同じ仕組みが使い回せることに気づけるかどうかが重要です。

演習④-A：Organismsを作る ― ヘッダー

ヘッダー全体のコンポーネント化

1. WFページで Header フレーム全体を選択

2. コピーして Componentsページに貼り付け

3. Ctrl+Alt+K → 名前を Organism/Header に変更

4. WFページに戻り、元のHeaderを削除 → アセットパネルから Organism/Header のインスタンスを配置

確認ポイント

ヘッダー内のCTAボタンが Button/Primary のインスタンスであることを確認してください。Organismsの中にAtomsのインスタンスが含まれている ＝ Atomic Designの入れ子構造です。

5ページ共通化のメリット

SŌILサイトが5ページ構成だとすると、ヘッダーの修正がメイン1回で5ページ分反映されます。ナビの項目追加、CTAボタンの色変更、ロゴの差し替え——すべてComponentsページだけで完結します。

⚠️ よくある間違い：WFページの要素をコンポーネント化した後、Componentsページに移動せずそのまま作業を続けてしまう。メインは必ずComponentsページに移動させましょう。

演習④-B：Organismsを作る ― 商品一覧セクション

Section_03_Products全体のコンポーネント化

1. WFページで Section_03_Products フレーム全体を選択

2. コピーして Componentsページに貼り付け

3. Ctrl+Alt+K → 名前を Organism/ProductList に変更

4. WFページの元のセクションをインスタンスに差し替え

入れ子構造を目で確認する

レイヤーパネルで Organism/ProductList を展開してみてください：

  • Organism：ProductList（セクション全体）
  • 　├ Atom：Heading/Section（見出し）
  • 　├ Molecule：Card/Product（商品カード×3）
  • 　│　└ Atom：Button/Secondary（ボタン）

Atoms → Molecules → Organisms が実際に入れ子で積み上がっているのが見えます。これが理論編で学んだ「小さい部品から大きい構造へ」の実体です。

🏗️ この瞬間が最重要：理論編の図解が、自分のFigmaファイルの中で「実物」として存在していることを確認してください。

セルフチェック＋まとめ

完成チェックリスト

  • ☐ 命名規則が カテゴリ/種類 で統一されているか
  • ☐ メインコンポーネントがComponentsページに集約されているか
  • ☐ WFページの要素がインスタンスに差し替えられているか
  • ☐ バリアント（Button/Primary ↔ Secondary）が正しく切り替わるか

動作確認テスト

ComponentsページのButton/Primaryの高さを48 → 56に変更してみてください。WFページのすべてのボタンインスタンスが自動で高さ56に更新されるはずです。確認できたら48に戻してください。

今回の成果

011で手作業で並べたWFの要素を、Atomic Designの層に沿って再構築しました。同じ見た目のWFでも、裏側の構造がまったく違います。
