LP設計：要件定義から
WFへの落とし込み実践【理論編】

Block 1–4のすべてを、1枚のLPに統合する

Block 5の全体地図——8テーマ・55コマの流れを一望する

Block 4で手元に何があるか？——素材の棚卸し

実践編に入る前に、すでに持っている素材を確認する。「これから新しく作る」のではなく、「持っている素材を設計で組み立てる」フェーズに入る。

テキスト系素材

  • 017：ペルソナ別ベストコピー9案・セクション見出し・CTA文
  • 016：3ペルソナ分のプロンプト設計シート（ペルソナ情報として活用）
  • 020：Gem「TALENTBOARDコピーライター」（不足テキストの追加生成に使用）

ビジュアル系素材

  • 018–019：統一トンマナの画像セット5枚＋ビジュアルガイドライン
  • 021：著作権チェック済み素材一式（使用可否が確定済み）
💡 素材はすべて揃っている。今日の仕事は「何を・誰に・どの順番で」という設計を決めること

「素材がある」と「LPが作れる」は違う

素材をただ並べると何が起きるか

  • ターゲットが曖昧なまま「全員向け」の文章を並べる
  • 訴求軸がブレてコピーが矛盾する
  • 情報の順番に根拠がなく、読者がCTAに到達しない
  • 結果：素材の品質に関係なく、成果が出ない

Block 001を思い出す

  • キャリアリンクファクトリーの1回目LP——素材は揃っていたが成果が出なかった
  • なぜか？「製造業全般に"何でもできます"」という設計のなさ
  • 001で学んだ原点：デザイン＝設計
  • 見た目を作る前に「設計」がある
📌 今日のテーマは「設計」。素材の品質が高くても、設計がなければ機能しない

LP要件定義の3ステップ——Block 2をLP文脈に翻訳する

Block 2で学んだ要件定義の考え方を、このLPで「実際にどう決めるか」に落とし込む。

3ステップの全体像

  • ステップ1：ターゲットの確定
    3ペルソナのうち、このLPのメインターゲットは誰か？
  • ステップ2：訴求軸の確定
    メインターゲットに最も刺さるメッセージは何か？
  • ステップ3：情報設計の確定
    どの順番で伝えれば、ターゲットがCTAに到達するか？

Block 2との違い

  • Block 2は「概念」を学んだ——ペルソナとは何か・要件定義の考え方
  • 022は「実行」する——このLPで実際にどう決めるか
  • 抽象論ではなく、TALENTBOARDの具体に落とす
  • 決定した内容が WF に直接反映される
💡 ステップ1→2→3は順番通りに進める。ターゲットが決まらないと訴求軸は決められない。訴求軸が決まらないと情報の順番は決められない

ステップ1：メインターゲットを1人に絞る

TALENTBOARDの3ペルソナ

  • 人事部長（50代・決裁者）
    課題：採用コスト・リスク管理。関心：ROI・ガバナンス
  • 現場リクルーター（30代・実務者）
    課題：候補者管理の煩雑さ。関心：効率化・ミス撲滅
  • 経営層（40〜50代・ROI判断者）
    課題：採用の可視化。関心：投資対効果・スピード

選定の考え方

  • 1つのLPで全員に刺そうとすると、誰にも刺さらない
  • 001の失敗例（「製造業全般に何でもできます"」）の再現になる
  • メインターゲットを1人に絞り、サブターゲットを設定する
  • サブターゲットへの配慮は「読まれても違和感ない程度」に留める
📌 「誰に言わないかを決めること」が設計の第一歩。全員向けは誰も向けていない

ステップ2：訴求軸を設計する——課題と理想のギャップを埋める

Block 2のペルソナ設計 ＋ Block 4のLogic Promptの3層構造を接続し、「最も刺さるメッセージ」を1本に絞る。

訴求軸の設計式

  • 課題（現在地）：今、メインターゲットが感じている痛み
  • 理想（目指す姿）：ターゲットが本当に欲しいゴール状態
  • ギャップ：課題→理想を埋めるTALENTBOARDの価値
  • 訴求軸：ギャップを一言で言語化したキャッチライン

TALENTBOARDの例（リクルーター向け）

  • 課題：候補者管理がExcelで限界——対応漏れ・二重連絡が発生
  • 理想：一画面で全候補者の進捗が見える
  • ギャップ：情報の散在→一元管理
  • 訴求軸：「Excel地獄から、1クリックで全体把握へ」
💡 訴求軸が決まると、コピー9案の中から「使う1案」が自然に決まる。迷いがなくなる

ステップ3：LPの導線設計——セクション構成に「理由」を持たせる

BtoB LPの「型」

  • FV：3秒で価値を伝える
  • 課題提起：読者の痛みに共感する
  • 解決策：TALENTBOARDがどう解決するか
  • 機能紹介：具体的にできること
  • 信頼構築：導入実績・数字・お客様の声
  • CTA：今すぐ行動させる

なぜこの順番か——Block 1（005）と接続

  • 人間の意思決定プロセスそのもの
  • 課題認識 → 解決策の検討 → 信頼の確認 → 行動
  • この順番を破ると読者が離脱する
  • 「型」は縛りではなく、人間心理に基づく最適解
📌 「課題→解決→信頼→行動」は行動心理学の意思決定プロセス。LP構成はその設計図

セクション構成表——実践編で記入するフォーマット

各セクションに対して3つの要素を定義する。このフォーマットがWF設計の起点になる。

定義する3要素

  • 目的：このセクションは何を達成するか
    例：FV → 「3秒で価値命題を伝え、続きを読む気にさせる」
  • 配置コンテンツ：テキスト・画像・CTAの種類と量
    例：FV → 「キャッチコピー1文＋サブコピー2行＋CTA1個＋画面モック1枚」
  • 使用素材：Block 4のどの素材を割り当てるか
    例：FV → 「017のコピーB案＋018の画像No.2」

記入の順番

  • Step 1：セクション名を列挙（FV〜CTA）
  • Step 2：各セクションの「目的」を1文で書く
  • Step 3：配置するコンテンツ要素をリストアップ
  • Step 4：使用する素材を016〜021から選ぶ
  • Step 5：Gemの「TALENTBOARDコピーライター」で不足テキストを補う

WFとは何か——何を決めて、何を決めないか

Block 3（011）のFigma WF構築を振り返り、LP文脈での役割を明確化する。

WFで決めること

  • 情報の優先順位：上にあるものほど重要。何を最初に見せるか
  • セクションの面積配分：重要度＝画面占有率。FVに何%割くか
  • CTAの配置ポイント：どのタイミングで行動を促すか
  • PC→SP変換方針：2カラムをどう折り返すか

WFで決めないこと

  • 色・フォントの選定 → 026で扱う
  • 画像の細部・実際の素材 → 026〜027で扱う
  • アニメーション・インタラクション → 027で扱う
  • 実装コードの詳細 → 028で扱う

WFは「骨格」。肉付けは後の演習で行う。今は構造の正確さだけに集中する

⚠️ WFに色を付けたくなったら要注意。それは「見た目で決めている」証拠——設計が骨格を失う

BtoB LP WFの参考パターン——TALENTBOARDの骨格

PC版・SP版のWF設計方針——Block 3のレスポンシブ設計を接続

LP特有のSP設計ポイント

  • FV：テキスト＋画像は縦積み。テキストを上・画像を下
  • 2カラム→1カラム変換：コンテンツの優先順位で上から並べ直す
  • CTAボタン：サムゾーン（親指が届く画面下部）内に配置
  • 追従CTA：スクロールしても常に表示されるCTA帯の検討

022の進め方

  • まずPC版WFを完成させる（構造の骨格を確定）
  • その後SP版に変換する（PC→SPの変換方針を明示）
  • Block 3（013）で学んだ「コンテンツ優先・グリッド変換」を適用
  • SP版も023〜028の演習でそのまま使用する

SP比率が高い場合でも「PCファースト設計 → SP変換」の順は崩さない。構造の整合性を保つため

まとめ：理論編で設計図の骨格が決まった

理論編で確認したこと

  • Block 4の持ち込み素材の全量
  • LP要件定義の3ステップ（ターゲット→訴求軸→情報設計）
  • BtoB LPのセクション構成の「型」と論理的根拠
  • WFで決めること・決めないことの境界線
  • PC版→SP版の変換方針

実践編でやること

  • 演習2：セクション構成表の作成（素材の割り当て）

🗂️

素材の棚卸し

Block 4で揃えたコピー9案・画像セット・Gemを把握する。「これから作る」ではなく「すでにある素材を設計で組み立てる」フェーズへ

🎯

LP要件定義3ステップ

①ターゲット確定（1人に絞る）→ ②訴求軸の設計（課題と理想のギャップ）→ ③情報設計（人間の意思決定順）。この3ステップが設計の骨格

🏗️

WFの役割と境界線

WFで決めるのは「情報の優先順位・面積・CTAポイント・PC→SP方針」。色・フォント・画像の細部はWFの仕事ではない

022が全体のベース
