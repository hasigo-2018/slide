Logic Prompt：戦略をAI指示に翻訳する技術【理論編】

「何を・誰に・なぜ」を構造化すれば、AIの出力は劇的に変わる

AIは賢い。でも、指示が曖昧なら曖昧な答えしか返さない

💬 問い：「"いい感じのキャッチコピー作って"で、満足な出力が返ってきたことはありますか？」

よくある現実

  • AIに聞いたら当たり障りのない回答しか返ってこない
  • 何度も再生成しても似たような出力が繰り返される
  • 「もっとうまく使えたら…」と感じながら諦める

Block 4で学ぶこと

  • AIを"使う"から"操る"へ
  • プロンプトも"設計"である（Block 1「デザインは設計」と同じ原則）
  • 構造で書けば、AIは意図通りに動く
🔑 核心：AIの出力品質は、指示の構造で決まる

プロンプトの3層モデル：良い指示には必ず3つの層がある

Contextの設計：「誰に・何のために」をAIに伝える

⚠️ Contextが曖昧だと：AIは「誰に向けて書いているかわからない」状態になり、汎用的な答えしか返せない

Block 2との接続

Block 2で作ったペルソナの「誰に」が、そのままContextに入る

Contextに含める4要素

  • ターゲット：誰に向けた内容か
  • ターゲットの課題：何に悩んでいるか
  • 商品・サービス概要：何を提供するのか
  • 使用場面：どこで・いつ使われるか

ケーススタディ：TALENTBOARD

📌 人材紹介会社向けクラウド型採用管理ツール
コンセプト：「採用の属人化を、チームの仕組みに。」
Block 4〜5を通して使い続ける題材

❌ Bad Context：「採用管理ツールのコピーを書いて」

✅ Good Context：「人材紹介会社の人事部長（50代・採用コスト削減が最重要課題）に向けて、候補者管理を効率化するクラウドツール TALENTBOARDの価値を伝えるコピーを書いて」

InstructionとConstraintの設計：形式と制約を決める

② Instruction（指示）

  • 出力形式：コピー / 本文 / 箇条書き
  • 出力数：3案 / 5案 など
  • トーン
    BtoB → 信頼・論理的
    BtoC → 共感・親しみ

「どんな形で出してほしいか」を明確にすることで、AIの出力がフォーマットとして安定する

③ Constraint（制約）＝ガードレール

  • 禁止事項：誇大表現NG、競合名NG
  • 文字数制限：20文字以内、など
  • 必須要素：サービス名・特定キーワードを含む
🔒 Constraintがないと：
AIは自由に暴走する。「採用革命」「圧倒的な」など誇大表現が混入する

4段階比較で「構造の差」を体感する

📊 対象：TALENTBOARD 人事部長向けキャッチコピー生成プロンプト

4段階の構造

  • レベル1：丸投げ型（3層なし）
  • レベル2：＋Context
  • レベル3：＋Context ＋ Instruction
  • レベル4：＋Context ＋ Instruction ＋ Constraint（完全版）

見るべきポイント

  • 出力にペルソナの関心語が含まれているか
  • フォーマットと文字数が指定通りか
  • 誇大表現や抽象表現が混入していないか

「構造化は、センスではなく設計力」

レベル1 → レベル2：Contextを追加すると何が変わるか

レベル1：丸投げ

📝 プロンプト：「採用管理ツールのキャッチコピーを3つ書いて」

出力例

  • 「採用をもっとスマートに」
  • 「ベストな人材を、最短で」
  • 「採用の未来を変えるツール」

→ 汎用的・誰向けか不明・どのツールでも使えそうな内容

レベル2：＋Context

📝 プロンプト：「人材紹介会社の人事部長（50代・採用コスト削減が最重要課題）に向けた、候補者管理クラウドツール TALENTBOARDのキャッチコピーを3つ書いて」

出力例

  • 「採用コストを削減する、チームの仕組みを作ろう」
  • 「属人化した採用を、組織の資産に変える」
  • 「コスト・効率・見える化。採用改革の第一歩」

→ 「コスト」「効率化」「属人化」などペルソナの関心語が出現

レベル3 → レベル4：InstructionとConstraintを加えると何が変わるか

レベル3：＋Instruction

📝 追加指示：「BtoBトーンで・20文字以内・数値や具体的業務改善を含む形式で3案」

出力例

  • 「採用コスト30%削減。仕組みで実現」
  • 「候補者管理、Excel卒業。採用DXの第一歩」
  • 「採用ROIを数字で証明するツール」

→ フォーマット安定・トーン一致。ただし「DX」「第一歩」など抽象語が残る

レベル4：完全版（＋Constraint）

📝 追加制約：「誇大表現・抽象語（DX・革命・最高・圧倒的）禁止。「TALENTBOARD」を必ず含める。具体的な業務改善を示すこと」

出力例

  • 「採用の属人化を、TALENTBOARDがチームの仕組みに変える」
  • 「面接履歴・評価・連絡をTALENTBOARDで一元管理」
  • 「TALENTBOARD：採用コスト削減を数字で見せるツール」

→ 具体性・一貫性・ブランド整合性が格段に向上

構造化で出力品質はここまで変わる

ビジュアルアーキテクトの仕事：ビジネスの意図をAI指示に翻訳する

実演：TALENTBOARDの3ペルソナ × Contextの違い

同じサービスでも、ペルソナが変わればContextが変わり、AIの出力トーンも変わる

👔 人事部長（50代・決裁者）

年間採用コスト削減と早期離職防止が最大の関心事。数値と根拠で説得したい

Context強調語：採用コスト削減・ROI・チーム標準化

📋 現場リクルーター（30代・実務者）

候補者50名以上を同時管理。Excel・メール・チャットの散在に疲弊している

Context強調語：業務効率化・一元管理・手間削減

📈 経営層（40〜50代・ROI判断者）

採用KPIの可視化と投資対効果のデータが必要。意思決定の根拠を求める

Context強調語：KPI可視化・データドリブン・投資対効果

🔑 結論：ペルソナ設計の精度が、プロンプト品質の上限を決める

AI出力を「なんとなく良い」で済ませない：評価の3軸

✅ 評価の問い：この出力は、本当に「そのペルソナ」に刺さっているか？

① ペルソナ適合度

ターゲットの課題・関心キーワードが出力に反映されているか

人事部長向けなら「コスト」「仕組み化」が含まれているか

② 具体性

抽象的な美辞麗句ではなく、具体的なベネフィットが示されているか

「革新的な」→ ✗　／　「候補者管理を一元化してコストを削減」→ ✓

③ トーン一貫性

BtoBで指定したのに感情的・BtoC的な表現が混入していないか

「楽しい採用体験を！」→ BtoBトーン違反

🔗 Block 6（論理的言語化）への伏線：
この3軸評価は、クライアントへの提案・レビューで使う「根拠の言語化」にそのまま活用できる

プロンプトは Context / Instruction / Constraint の3層で設計する。どれか一つでも欠けると出力はブレる

⚖️

② 構造の有無が差を生む

Bad → Goodの差はセンスではなく設計力。4段階で出力品質は20%から90%まで変わる

👤

③ ペルソナがContextを決める

ペルソナが変わればContextが変わり、出力が変わる。Block 2の上流設計がBlock 4全体を支える

💡 プロンプトエンジニアリングは特殊スキルではない。構造で考える習慣があれば誰でも身につけられる
