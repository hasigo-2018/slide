## 1. 環境セットアップ（VS Code版）

### 1-1. Claude Code 拡張機能のインストール

1. VS Codeを開く
2. `Cmd + Shift + X` で拡張機能パネルを開く
3. 「Claude Code」で検索（発行元: Anthropic）
4. 「Install」をクリック
5. インストール後、左サイドバーに ✦（Sparkアイコン）が表示される
6. 初回起動時にAnthropicアカウントでサインインを求められるので、ログインする

### 1-2. プロジェクトフォルダを開く

1. VS Codeで `ファイル > フォルダーを開く` を選択
2. `/Users/akihirotakahashi/Documents/slide` を開く
3. フォルダ構成が以下のようになっていることを確認：

```
slide/
├── CLAUDE.md
├── generate.sh
├── fix-fontsize.sh
├── scripts/
│   ├── insert_html.js
│   └── check_svg.js
├── master_slide/
│   └── themes_new/
│       ├── 014-1_theory.html
│       ├── 014-2_practice.html
│       └── ...
├── 教科書の構成指示書/
│   ├── block 1/
│   ├── block 2/
│   └── ...
└── assets/
    ├── slide-template.css
    └── slide-template.js
```

### 1-3. ターミナルの使い方（VS Code内）

VS Code 内でターミナルを開く方法：
- `` Ctrl + ` ``（バッククォート）を押す
- または メニューの `ターミナル > 新しいターミナル`

ターミナルが開いたら、自動的に `slide` フォルダがカレントディレクトリになっています。

## 2. 教科書生成の手順

### 2-1. 起動方法

VS Code のターミナルで以下を実行：

```bash
./generate.sh 014
```

※ 番号は生成したいテーマ番号（001〜044）に変更してください。

### 2-2. 処理の流れ

```
./generate.sh 014 を実行
    ↓
ファイル存在チェック（自動）
    ↓
Claude Code が起動
    ↓
━━━ 理論編 ━━━
    ↓
① 構成指示書を読み込み、方向性・確認事項を提示
    ↓
② AKIさんが確認 → 承認 or 修正指示
    ↓
③ 理論編HTMLを生成
    ↓
④ 雛形HTMLに自動挿入（insert_html.js）
    ↓
⑤ SVGチェック実行（check_svg.js）→ 結果をレポート
    ↓
⑥ AKIさんが確認 → 「理論編OK」と入力
    ↓
━━━ 実践編 ━━━
    ↓
⑦ 「実践編に進みますか？」→ 「はい」
    ↓
⑧〜⑫ 理論編と同じ流れ
    ↓
⑬ 「実践編OK」→ 完了
```

### 2-3. 確認フローでの操作

| やりたいこと | 入力する言葉 |
|-------------|-------------|
| 理論編を承認して次へ | `理論編OK` |
| 実践編を承認して完了 | `実践編OK` |
| 特定スライドを修正 | `スライド8を修正：矢印を右向きに変えて` |
| 理論編に戻る | `理論編に戻って` |
| 最初から作り直す | `理論編を最初から` |
| 全スライドを再出力 | `全体を出力して` |

### 2-4. 1日の推奨ペース

- **2〜3冊/日**（理論編+実践編 = 4〜6ファイル）
- Proプランで無理なく収まる範囲

## 3. 文字サイズ修正の手順

### 3-1. 概要

既に生成済みのHTMLファイル（001〜014）について、インラインスタイルの `font-size` をCSSクラスに置換し、文脈に応じた適切な文字サイズに調整します。

### 3-2. 前提作業（初回のみ）

`slide-template.css` の末尾に以下を追記してください：

```css
/* テキストサイズ・ユーティリティクラス */
.text-sub {
    font-size: 26px;
    line-height: 1.6;
    color: #e0e0e0;
}

.text-note {
    font-size: 21px;
    line-height: 1.6;
    color: #999;
}
```

### 3-3. 起動方法

VS Code のターミナルで以下を実行：

```bash
# 1冊分（理論編+実践編の2ファイル）を調整
./fix-fontsize.sh 001

# 範囲指定（001〜013を連続処理）
./fix-fontsize.sh 001 013
```

### 3-4. 処理の流れ

```
./fix-fontsize.sh 001 を実行
    ↓
Claude Code が起動
    ↓
① HTMLファイルを読み込み
    ↓
② インラインの font-size を検出
    ↓
③ 文脈を判断して適切なクラスに置換
    ・メインの説明文 → 32px（デフォルト、クラスなし）
    ・補足的な説明  → 26px（.text-sub）
    ・注釈・出典    → 21px（.text-note）
    ↓
④ 修正箇所のレポートを表示
    ↓
⑤ AKIさんが確認 → OKなら次のファイルへ
```

### 3-5. 文字サイズの階層

| 用途 | サイズ | 指定方法 |
|------|--------|---------|
| タイトル h1 | 72px / 112px | CSSで自動適用 |
| セクション h2 | 56px〜96px | CSSで自動適用 |
| 見出し h3 | 40px | CSSで自動適用 |
| 本文 p, li | 32px | クラス指定なし |
| 小さめ本文 | 26px | `class="text-sub"` |
| 補足・注釈 | 21px | `class="text-note"` |
| highlight-box | 24px | CSSで自動適用 |

**21px未満の文字サイズは一切使用禁止。**

### 3-6. 推奨ペース

- 空き時間に数冊ずつ進めればOK
- 1ファイルあたり数分で完了

## 4. トラブルシューティング

### 「テーマ番号が範囲外」と出る
→ 001〜044 の範囲で指定してください

### 「構成指示書が見つかりません」と出る
→ `教科書の構成指示書/block N/` 内のファイル名を確認。`blueprint_XXX_theory.md` の形式になっているか

### 「雛形HTMLが見つかりません」と出る
→ `master_slide/themes_new/XXX-1_theory.html` の存在を確認

### Claude Code が途中で止まる
→ Proプランの使用量上限に達した可能性。時間を置いて再試行

### 挿入前のHTMLに戻したい
→ `XXX-1_theory.html.bak` というバックアップファイルが自動生成されています。元のファイル名にリネームしてください

### VS Code でClaude Codeが表示されない
→ 拡張機能パネルで「Claude Code」がインストール済みか確認。インストール後にVS Codeを再起動してください
