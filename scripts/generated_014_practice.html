プロトタイピングと動的デモ【実践編】

SŌILのWFを"触れるデモ"にする

演習①-準備：下層ページのダミーフレーム作成

📐 プロトタイプで遷移させるには、遷移先のフレームが存在している必要がある。まず最小限のダミーページを5枚用意する

作成するダミーフレーム（5枚）

  • About（1440px幅）
  • Products（1440px幅）
  • Story（1440px幅）
  • Contact（1440px幅）
  • Shop_External「外部サイトへ遷移します」のダミー1枚

各ダミーに必要な3要素

  • ① Organism/Header のインスタンスを配置
    012のコンポーネントを再利用 → ヘッダー修正が全ページに自動反映
  • ② ページタイトル（テキストのみでOK）
  • ③ 「TOPに戻る」テキストリンクを配置

中身の作り込みは不要。遷移先として「存在すること」が目的

演習②-Step 1：ヘッダーナビの画面遷移を設定する

🔧 Figmaの Prototypeタブ に切り替え → ヘッダーナビの各項目からドラッグして接続
  • 接続する項目： About / Products / Story / Contact / Shop（5本）
  • トリガー： On Click
  • アクション： Navigate to → 対応するダミーフレーム
  • トランジション： Dissolve / 300ms

接続の設定手順

  1. ナビ項目テキストを選択
  2. 青い丸（ハンドル）が右端に出現
  3. ドラッグして遷移先フレームにドロップ
  4. 右パネルでアクション・トランジションを確認

ポイント

  • コンポーネントインスタンス内の要素に接続する場合は、インスタンスをダブルクリックして内部要素を選択
  • 接続線は青い矢印で表示される
  • 5本すべて設定できたか確認してから次へ

演習②-Step 2：各ページからTOPへの戻り導線を設定する

🔄 プロトタイプで行き止まりを作らないために、すべてのダミーページからTOPに戻れる導線を設定する

About / Products / Story / Contact

  • 各ダミーフレームのヘッダーロゴ
    → On Click → Navigate to → TOPフレーム
    Dissolve / 300ms
  • 「TOPに戻る」テキストリンク
    → 同じ設定でOK

Shop_External（外部遷移ダミー）

  • 「戻る」ボタン
    → On Click → Back
    アクションは Back を選択。前の画面に戻る
✅ 往復確認： TOP → About → TOP → Products → TOP の順にクリックして、すべて戻れることを確認してから次の演習へ

演習③：TOPページ内のセクション導線を設定する

  • Section_02_About「もっと見る」→ Navigate to → About
  • Section_03_Products「すべて見る」→ Navigate to → Products
  • Section_04_Story「ストーリーを読む」→ Navigate to → Story
  • Section_05_Instagram「フォロー」→ Navigate to → Shop_External
  • Section_06_CTA「オンラインショップへ」→ Navigate to → Shop_External
  • Footer ナビ → 各項目をヘッダーナビと同じ遷移先に設定
アクションの使い分け整理
同じページ内のセクション移動 → Scroll to（今回のTOPページ内スクロール）
別ページへの遷移 → Navigate to
外部サイトへの遷移（WF段階）→ ダミーフレームへ Navigate to

演習④-Step 1：ドロワーメニューのフレームを作成する

💡 013で保留していた「ハンバーガーメニュー展開」を、ここで Open Overlay として完成させる

フレームのスペック

  • フレーム名：Drawer_Menu
  • 幅：300px ／ 高さ：812px
  • 背景：濃いダーク系（TOP背景と統一）

内部レイアウト

  • Auto Layout：Vertical / Gap 16px / Padding 24px
  • 上部に「✕」閉じるボタンを配置
  • ナビ項目を縦積み（About / Products / Story / Contact / Shop）
  • 各ナビ項目に Navigate to → 対応ダミーフレームを設定
📝 補足： 今回はPC版のWF上での練習。SP版ハンバーガーでもまったく同じ手法（Open Overlay）を使うため、Block 5で即応用できる

演習④-Step 2：Overlayの接続を設定してドロワーを動かす

ハンバーガーアイコンの設定

  • 接続元：ヘッダーのハンバーガーアイコン
  • トリガー：On Click
  • アクション：Open Overlay
  • 遷移先：Drawer_Menu フレーム
  • 位置：Top Right
  • アニメーション：Move in from Right / 300ms
  • 背景クリックで閉じる：ON

「✕」ボタンの設定

  • 接続元：Drawer_Menu内の✕ ボタン
  • トリガー：On Click
  • アクション：Back

Back を使うことで「Overlayを閉じて元の画面に戻る」動作が再現される

▶ プレビューで動作確認： ハンバーガーをクリック → 右からドロワーがスライドイン → ナビ項目でページ遷移 → ✕ or 背景クリックで閉じる

プレビュー確認＋セルフチェック

プレビューの起動方法

  1. 右上の ▶（Present） ボタンをクリック
  2. Flow の開始地点 ＝ TOPページ
  3. ブラウザが開き、操作可能な状態になる
  4. Share → Copy link でプレビューURLを取得

セルフチェックリスト

  • ☐ ナビ遷移が全ページ（5接続）正しく動くか
  • ☐ 各ページからTOPに戻れるか
  • ☐ 各セクションのリンクが正しい遷移先か
  • ☐ Overlayの開閉が自然に動くか
  • ☐ トランジションがすべて統一されているか
  • ☐ 行き止まりのページが存在しないか
🔗 取得したプレビューURLをコピーして、ブラウザで操作確認。Figmaアカウントがなくても閲覧・操作できる

まとめ：3つのアクションで「触れるデモ」が完成した

今回使った3アクション

  • Navigate to
    ページ間遷移。ナビ・CTA・セクション導線
  • Open Overlay
    ドロワーメニュー。画面の上に重ねて表示
  • Back
    Overlay閉じる＆外部ページから戻る

達成したこと

  • コードなしで「触れるデモ」を作成
  • 013の保留事項（ドロワー）を回収・完成
  • 012コンポーネントをダミーページで再利用
  • プレビューURLでクライアントへ共有可能
