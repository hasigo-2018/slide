# 構成指示書：022 LP設計：要件定義からWFへの落とし込み実践【実践編】

## 基本情報
- **テーマ:** LP設計：要件定義からWFへの落とし込み実践
- **目的:** TALENTBOARDのLP要件定義書・セクション構成表を作成し、Figmaでワイヤーフレーム（PC版・SP版）を完成させる。このWFが023〜028の全演習のベースとなる
- **推奨スライド数:** 17〜21枚
- **推奨時間:** 約50分×6コマ（300分）
- **使用ツール:** Figma、Gemini（Gem「TALENTBOARDコピーライター」）
- **ケーススタディ:** TALENTBOARD（人材紹介会社向けクラウド型採用管理ツール）

## 重要な設計意図
- Block 5で最もコマ数の多い実践（6コマ）。022のWFがブレると023〜028の全演習が崩れるため、時間をかけて丁寧に完成させる
- 演習を3段階（要件定義書→セクション構成表→Figma WF）に分け、「考える→決める→形にする」の順序を守らせる。いきなりFigmaを開かせない
- Block 2（上流設計）とBlock 3（Figma）の知識を「LP制作」という出口で初めて統合する場。受講生にとっては「今まで学んだことが全部つながる」体験
- 001実践編の「100文字で言語化する」訓練を継続。各演習の最後に「なぜそう決めたか」を短文で書かせる
- Block 4のGem（TALENTBOARDコピーライター）を実務ツールとして初めて使う場面を設計。不足テキストはGemで追加生成する
- Block 2（整骨院）との接続を演習開始前に明示する：スライド3で受講生に整骨院の要件定義書を手元に出させ、「同じフレームをTALENTBOARDに転用する」ことを宣言する。整骨院が「習熟」、TALENTBOARDが「実戦」という位置づけを受講生が自覚することで、Block 2の学習が「使われなかった練習」にならない

## クイズ（5問・実践判断型）

| No. | 問題 | 正解のポイント |
|-----|------|---------------|
| Q1 | LP要件定義書で最初に確定すべきことは？ | メインターゲット（誰に向けたLPか）。訴求軸もセクション構成もターゲットが決まらないと定義できない |
| Q2 | 3ペルソナがいるとき「全員に刺さるLPを作ろう」としたらどうなるか？ | 001のキャリアリンクファクトリー1回目と同じ失敗。ターゲットが広すぎて誰にも刺さらない |
| Q3 | セクション構成表で各セクションに「目的」を書く理由は？ | 「なんとなくここに置いた」を排除し、セクションの存在理由を言語化するため。Block 6の論理的言語化の下地 |
| Q4 | WFを作る前に要件定義書とセクション構成表を先に作る理由は？ | 「何を・誰に・どの順番で伝えるか」が決まっていない状態でWFを組むと、見た目の都合で情報設計がゆがむ |
| Q5 | Gem「TALENTBOARDコピーライター」を022の演習で使う場面はどこか？ | セクション構成表の作成時に、Block 4で未生成のテキスト（課題提起本文・FAQ等）を追加生成する場面 |
