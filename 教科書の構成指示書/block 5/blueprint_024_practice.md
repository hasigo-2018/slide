# 構成指示書：024 信頼構築のセクション設計【実践編】

## 基本情報
- **テーマ:** 信頼構築のセクション設計
- **目的:** TALENTBOARDの信頼構築セクション（数字実績・導入事例Before/After・お客様の声・ロゴ帯）をFigmaでデザインし、セクション間の導線が一貫していることを確認する
- **推奨スライド数:** 16〜19枚
- **推奨時間:** 約50分×5コマ（250分）
- **使用ツール:** Figma
- **ケーススタディ:** TALENTBOARD（人材紹介会社向けクラウド型採用管理ツール）

## 重要な設計意図
- 信頼構築セクションは4つの型（数字実績・導入事例・お客様の声・ロゴ帯）があり、実践5コマで全型を制作する。演習1（数字実績 / 1.5コマ）→ 演習2（導入事例Before/After / 2コマ）→ 演習3（お客様の声＋ロゴ帯＋導線確認 / 1.5コマ）
- 演習2（導入事例Before/After）に最も多くのコマを割く理由：Before/Afterは「物語構造＋視覚的対比」が必要で、情報量も最大のセクション。3社分を作り、レイアウトのバリエーション（カード型・タブ切り替え型・1社フル展開型）を試す
- 架空データは理論編の構成指示書に記載済み。受講生はデータを「素材」としてそのまま使い、「どう見せるか」のデザインに集中する
- 各演習の最後に「なぜこのレイアウトを選んだか」を100文字で言語化する訓練を継続

## ケーススタディ設定（TALENTBOARD）

| 項目 | 内容 |
|------|------|
| サービス名 | TALENTBOARD（タレントボード） |
| 使用する架空データ | 理論編に記載の数字実績4指標＋導入企業3社の事例＋担当者コメント |
| 022で確定したWF | 信頼構築セクションの位置と面積は確定済み |
| 023で完成したFV | FVからの導線が信頼構築セクションに自然につながるか確認 |

## クイズ（5問・実践判断型）

| No. | 問題 | 正解のポイント |
|-----|------|---------------|
| Q1 | 導入事例のレイアウト（カード型・タブ型・フル展開型）を選ぶ判断基準は？ | 022で確定したメインターゲットにとってどの型が最も効果的か。一覧性重視ならカード型、詳細重視ならタブ型、バランスならフル展開型 |
| Q2 | Before/Afterの視覚的対比で色分けを使う理由は？ | Before（課題・痛み）とAfter（成果・改善）の変化を瞬時に認識させるため。テキストだけでは読まないと伝わらないが、色は見るだけで伝わる |
| Q3 | 1社目をフルデザインしてからコンポーネント化し2〜3社目に展開する利点は？ | Block 3（Atomic Design）の実践。1回の設計で複数の事例に展開でき、修正も一括で反映できる。実務のスピードと一貫性を担保する手法 |
| Q4 | ロゴ帯を信頼構築セクションの冒頭に置く効果は？ | 最も認知負荷が低い（ロゴを見るだけ）社会的証明。スクロールした瞬間に「これだけの企業が使っている」と瞬時に伝わり、後続の詳細セクションを読む動機を作る |
| Q5 | 導線の通し確認で「セクション間の余白」をチェックする理由は？ | 余白が狭すぎるとセクション間の区切りが曖昧になり、広すぎると導線が切れて離脱リスクが上がる。余白もデザインの一部 |
