# 構成指示書：029 【課題提出】LPデザイン案の完成と「100文字エビデンス」【実践編】

## 基本情報
- **テーマ:** 【課題提出】LPデザイン案の完成と「100文字エビデンス」
- **目的:** TALENTBOARDのLP一式を完成させ、全セクションに100文字エビデンスを付与し、ペアレビューを経て最終提出する。Block 5の全成果物を「LP一式＋エビデンス＋バナー群＋最適化画像群」としてパッケージ化する
- **推奨スライド数:** 20〜24枚
- **推奨時間:** 約50分×8コマ（400分）
- **使用ツール:** Figma、Gemini
- **ケーススタディ:** TALENTBOARD（人材紹介会社向けクラウド型採用管理ツール）

## 重要な設計意図
- 実践8コマを4段階に分ける。演習1（全セクション統合・通し確認 / 2コマ）→ 演習2（100文字エビデンス執筆 / 3コマ）→ 演習3（ペアレビュー＋修正 / 2コマ）→ 演習4（最終提出準備 / 1コマ）
- 演習2に3コマ（150分）を割く理由：8〜10セクション分のエビデンスを書くのは初学者にとって重い作業。1セクションあたり15分（考える10分＋書く5分）が目安。セクション数×15分＝120〜150分が必要
- 演習3のペアレビューはBlock 6（公開添削）のミニ版。「他者の作品を設計観点で読む」体験を029で入れておくことで、Block 6への心理的ハードルを下げる
- 演習4の最終提出準備で「LP一式＋エビデンス＋バナー＋画像」をパッケージ化する。この「まとめて渡す」作業はBlock 7（ハンドオフ）の伏線
- 001実践編で「100文字で書く」訓練を始め、023（FVベスト選定理由）→024（レイアウト選択理由）→027（バナーベスト3選定理由）と続けてきた言語化訓練の集大成。029で「全セクション分」書くことで量的な飛躍をさせる

## クイズ（5問・実践判断型）

| No. | 問題 | 正解のポイント |
|-----|------|---------------|
| Q1 | 統合の通し確認で「導線の一貫性」をチェックする際、最も注意すべき箇所は？ | セクション間の「接続」。前のセクションの結論が次のセクションの冒頭につながっているか。個別セクションの品質が高くても接続が切れていればLPとして機能しない |
| Q2 | 100文字エビデンスを書いて「手が止まる」場合、最も可能性が高い原因は？ | そのデザイン判断に根拠がない。「なんとなく」で作った部分はエビデンスが書けない。Block 1–4のどの知識に基づいているか振り返る必要がある |
| Q3 | ペアレビューでもらったコメントに「納得できない」場合、どうすべきか？ | 修正しなくてよい。ただし「なぜ修正しないか」の理由をエビデンスに追記する。ビジュアルアーキテクトは全てのフィードバックに従う人ではなく、根拠を持って判断する人 |
| Q4 | Figmaファイルの整理で「他者が開いても迷わない構成」を目指す理由は？ | Block 7（エンジニアへのハンドオフ）で実装者がファイルを受け取る前提。自分だけがわかる構成では実務で通用しない |
| Q5 | Block 5のまとめ「"なぜこう作ったか"を全セクションに語れるLP」が「ビジュアルアーキテクト」の仕事である理由は？ | 「作れるだけのデザイナー」は成果物を出すことがゴール。「ビジュアルアーキテクト」は設計根拠を語れることがゴール。エビデンスが書けるということは、設計に根拠があるということ。根拠があれば再現もできる |
