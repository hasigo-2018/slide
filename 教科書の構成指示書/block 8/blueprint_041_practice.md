# 構成指示書：041 エビデンスの記述と作品の論理的パッケージング【実践編】

## 基本情報
- **テーマ:** エビデンスの記述と作品の論理的パッケージング
- **目的:** TALENTBOARDの棚卸しシートを基にケーススタディを執筆し、過去作品も同じ型で変換して、ポートフォリオの骨格を完成させる
- **推奨スライド数:** 14〜16枚
- **推奨時間:** 約50分×3コマ（150分）
- **使用ツール:** Gemini（キャプション推敲補助）、Googleスプレッドシートまたは紙（棚卸しシート）、Figmaまたはテキストエディタ（ケーススタディ執筆）
- **素材:** TALENTBOARD LP一式＋040の棚卸しシート＋受講生の過去作品（ある場合）

## 重要な設計意図
- 実践は4段階で進める。①過去作品の再定義と棚卸しシートへの追加 → ②実績の選定と並び順の設計 → ③ケーススタディの執筆（TALENTBOARDメイン＋過去作品） → ④ペアレビューとキャプション推敲
- 041は040の棚卸しシートに「見せ方メモ」列を埋め、さらにケーススタディのテキストを実際に書き上げるテーマ。040が「素材の洗い出し」、041が「素材の構成と言語化」
- 過去作品は041で初めて合流する。完全未経験者はTALENTBOARD一本で進め、実務経験者はTALENTBOARD＋過去作品2〜3点で進める。構成指示書にはこの分岐を明記する
- ケーススタディの完成度は042（モックアップ）と043（自己ブランド）で上がるので、041ではテキストの骨格に集中する。ビジュアルの仕上げは042以降

## ケーススタディ設定（TALENTBOARD）

| 項目 | 内容 |
|------|------|
| サービス名 | TALENTBOARD（タレントボード） |
| 種別 | 人材紹介会社向けクラウド型採用管理ツール（ATS） |
| コンセプト | 「採用の属人化を、チームの仕組みに。」 |
| ターゲット | 50名以下の中小人材紹介会社の管理職（導入決裁者） |
| 040の成果物 | 棚卸しシート（LPの7セクション＋LP以外の成果物3件以上、各実績に「課題→施策→成果」記入済み） |

## クイズ（5問・実践判断型）

| No. | 問題 | 正解のポイント |
|-----|------|---------------|
| Q1 | ケーススタディの「施策」に「FVをデザインしました」と書いた場合、何が足りないですか？ | 設計判断の理由が欠けている。「FVにキャッチコピーと管理画面モックアップを配置し、Z法則に基づく視線誘導でCTAへ導く構成にした」のように、「何を・なぜそうしたか」まで書く |
| Q2 | 過去作品に定量的な成果データがない場合、「成果」欄にはどう書くべきですか？ | 定性的事実（クライアントの反応・リピート依頼の有無）を記載するか、「〜の心理学的効果により○○の改善が見込める」と仮説＋裏付けの形で記載する。根拠のない断言は避ける |
| Q3 | ペアレビューで「課題がデザイン用語になっている」と指摘された場合、具体的にどう修正しますか？ | 「レイアウトが見づらい」→「ターゲットである管理職が初見で"自社の課題が解決できる"と判断できない構成だった」のように、ビジネス上の問題として書き直す。読者はデザイナーではなくクライアントや採用担当者 |
| Q4 | ポートフォリオの1stにTALENTBOARD LP全体を置く理由は？ | コース内で最も深く設計した実績であり、要件定義→WF→デザイン→エビデンス→仕様書→ハンドオフまでの一貫したプロセスを証明できる。「課題→施策→成果」の論理が最も明確で、ビジュアルインパクトもある |
| Q5 | 041のケーススタディにビジュアルの「プレースホルダー」だけを入れ、実物を入れない理由は？ | 041はテキスト（骨格）に集中するテーマ。ビジュアルの制作は042（AIモックアップ）で行う。テキストとビジュアルを同時に作ろうとすると、どちらも中途半端になる。LP制作と同じで、WF（骨格）→ビジュアル（肉付け）の順で進める |
