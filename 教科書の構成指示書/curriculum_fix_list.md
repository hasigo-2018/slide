## 効率的な修正方法

### ステップ1：curriculum.pdfの更新（No.1, 8, 9）

最も効率的なのは、curriculum.pdfを1回の作業でまとめて更新すること。修正箇所は3つだけ。

- コマ配分の集計表：Block 8を15→20、バッファを15→10、合計を225→230に
- Block 7の欄に注記：「※並行カリキュラムでコーディング・チーム開発は学習済みの前提」
- 末尾に「並行する自習課題」セクションを追加：Figmaトレース（にんべん）、サイト模写、アーカイブ共有の存在を1段落で記載

**このチャットでcurriculum.pdfの更新版を作成可能。** 指示があれば対応します。

### ステップ2：Block 8の構成指示書確認（No.2, 3）

Block 8チャット（https://claude.ai/chat/4b94b826-08a7-4bf8-acdb-4da1000ee59d）を開き、以下を確認する。

- 043の構成指示書に「キャリア戦略・案件の狙い方」の要素があるか → なければ043のスコープを拡張
- 044のコマ数が模擬プレゼン＋フィードバック＋修正に十分か → 不足ならバッファからコマを追加

**確認→修正の判断→構成指示書の更新、を1チャット内で完結させるのが効率的。** Block 8チャットの続きとしてやるか、新規チャットでやるかはAKIの判断。

### ステップ3：Block 5 / 026の確認（No.4）

Block 5チャットを開き、026（AI画像加工）の構成指示書のコマ数・カバー範囲を確認する。旧5テーマの内容（背景拡張・不要物除去・ライティング・人物最適化・商用広告への昇華）が十分にカバーされているかチェック。

**ステップ2と同じ要領。** 確認して問題なければスキップ、不足があればバッファから充当。

### ステップ4：実例サイトリストの反映（No.6, 7）

Block 1とBlock 6の構成指示書に実例サイトURLを追記する作業。各チャットの構成指示書を開いて追記するか、一括で修正リストを作成してまとめて反映する。

**このチャットで「追記すべきテキスト」を生成し、各チャットに貼り付ける形が効率的。**

### ステップ5：Block 7 / 039の前提修正（No.5）

039の構成指示書の「重要な制約」セクションから「HTML/CSSのコーディングスキルは前提としない」を削除し、「並行カリキュラムでHTML/CSS/JavaScriptの基礎は学習済みの前提。実装を意識したハンドオフ設計に踏み込む」に差し替え。

**影響範囲が039の1ファイル（理論編）に限定されるので、ピンポイントで修正可能。**

### ステップ6：Block 3 / 012〜015の整合性確認（No.10）

Block 3チャットを開き、012〜015の構成指示書がBlock 4以降の「既知前提」と整合しているか確認する。特に015（Figma AI機能）→ 016（Logic Prompt）の接続。

**確認だけなので工数は小さい。** 問題があれば修正、なければ完了。
