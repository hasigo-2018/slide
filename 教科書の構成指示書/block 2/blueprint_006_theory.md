# 構成指示書：006 WEB制作の全体工程マップとアーキテクトの役割【理論編】

## 基本情報
- **テーマ:** WEB制作の全体工程マップとアーキテクトの役割
- **目的:** WEB制作の6工程を俯瞰し、ビジュアルアーキテクトの立ち位置を理解する
- **推奨スライド数:** 15〜18枚
- **推奨時間:** 約35分
- **前提知識:** Block 1（デザイン基礎・Figma操作）修了済み。WEB制作の全体工程は未学習。

## 重要な制約
- 各工程の深掘りは007〜010で行うため、006では「全体像の理解」と「自分の立ち位置」に集中する
- 図解（6工程マップ・チーム相関図）がこのテーマの核。SVG図解で丁寧に作る
- Block 1との接続：「今まで学んだことは全体のどこだったのか」を理解させる
- アーキテクトの定義はコース全体の根幹。言葉選びは慎重に

## クイズ（5問）

| No. | 問題 | 正解のポイント |
|-----|------|---------------|
| Q1 | WEB制作の6工程を正しい順番で並べてください | 企画→IA→デザイン→実装→公開→運用 |
| Q2 | 「デザインが良いのに成果が出ない」最も多い原因は？ | 上流工程（企画・IA）の設計不足 |
| Q3 | ビジュアルアーキテクトとデザイナーの最大の違いは？ | デザイナーは「作る人」、アーキテクトは「設計して判断する人」 |
| Q4 | AI時代にアーキテクトが重要になる理由は？ | AIが「作る」を代替→人間は「決める」「判断する」に集中する必要がある |
| Q5 | 工程間の受け渡しが雑だと何が起きる？ | 手戻り・認識のズレ→品質低下とスケジュール遅延 |
