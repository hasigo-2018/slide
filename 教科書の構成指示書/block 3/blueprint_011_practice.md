# 構成指示書：011 ワイヤーフレーム構築【実践編】

## 基本情報
- **テーマ:** ワイヤーフレーム構築
- **目的:** SŌILのブランドサイトTOPページのLo-Fi WFをFigmaで構築する
- **推奨スライド数:** 8〜10枚
- **推奨時間:** 約20分 + 演習40分
- **使用ツール:** Figma, ChatGPT
- **ケーススタディ:** SŌIL（Block 3共通）

## 重要な設計意図
- 011〜014の実践編はすべてSŌILのTOPページWFを段階的に進化させる一気通貫の構成
- Block 5の022ではSŌILの「新商品LP」のWFを作るため、011ではTOP型に集中して棲み分け
- ChatGPTは「TOPの構成要素の洗い出し」に使い、優先順位の判断は人間が行うパターンを確立（Block 3全体で継続）
- Block 2の上流設計（要件定義→ペルソナ→サイトマップ→トンマナ）が画面構造に直結することを体験させる

## 理論編からの踏襲ポイント
- WFは「何を・どこに・どの優先度で置くか」を決める設計図。今回作るのはLo-Fi WF
- TOP型は「選ばせる入口」を設計する。LP型の「読ませる順番」とは構造が異なる
- TOPページの5構成要素：ヘッダー＋ナビ／MV／コンテンツエリア／CTA導線／フッター
- サイトマップ＋ペルソナの行動目的からTOPの情報優先度を逆算する
- Figma上では「見た目を作らない」ルール（グレースケールのみ、装飾なし）

## ビジュアルアーキテクト育成上の位置づけ

このテーマは、題材「ワイヤーフレーム構築」を通じて、ビジュアルアーキテクトに必要な「目的に沿って設計・判断・説明する力」を鍛える位置づけである。単に作業手順やツール操作を覚えるのではなく、このテーマの目的（SŌILのブランドサイトTOPページのLo-Fi WFをFigmaで構築する）に向けて、何を選び、なぜそうするかを自分の言葉で説明できる状態をゴールとする。

## 視覚化方針

このテーマでは、文章説明だけでなく、必要に応じてSVG図解・画像生成・既存アセットを使い分ける。特に、初学者が状況を想像しづらい箇所、Before/After、判断基準、成果物イメージ、実務での使用場面は視覚化候補として扱う。スライド生成時は「画像があると理解が速くなるか」「SVGで構造化した方がよいか」「既存素材で具体化できるか」を確認し、視覚要素が学習目的を支える場合に採用する。
