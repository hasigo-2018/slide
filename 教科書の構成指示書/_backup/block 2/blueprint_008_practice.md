# 構成指示書：008 ペルソナ設計とユーザージャーニー【実践編】

## 基本情報
- **テーマ:** ペルソナ設計とユーザージャーニー
- **目的:** 統合型ペルソナシート（ペルソナ＋JTBD）と簡易ユーザージャーニーマップを作成する
- **推奨スライド数:** 10〜12枚
- **推奨時間:** 約20分 + 演習30分
- **使用ツール:** Gemini, Figma
- **実例:** 007から継続の整骨院案件

## 重要な設計意図
- 007からの連続演習。要件定義をペルソナのインプットにする
- Geminiプロンプトを2段構え：①ペルソナ生成（WHO） ②JTBD生成（WHY）。両者の違いを体感させる
- ジョブを3パターン出させることで、同じペルソナでも状況が変わればジョブが変わることを実感させる
- ジャーニーマップは最優先ジョブ1つに絞る（時間の都合＋「選ぶ」判断力の訓練）
- 009への接続：振り返りで挙げた「必要ページリスト」を保存させる

## クイズ（5問・実践判断型）

| No. | 問題 | 正解のポイント |
|-----|------|---------------|
| Q1 | Geminiでペルソナを生成した後、最初にチェックすべきことは？ | 平均的すぎないか、行動の理由が見えるストーリーになっているか |
| Q2 | JTBDを3パターン出す理由は？ | 同じペルソナでも状況が変わればジョブが変わることを理解するため |
| Q3 | ジャーニーマップで「感情」の行を埋める目的は？ | ユーザーの不安やストレスを特定し、サイトで解消すべきポイントを明確にするため |
| Q4 | ジャーニーの「比較検討」ステップから見えるサイトに必要なページは？ | 料金ページ、口コミ・実績ページ、他院との比較情報 |
| Q5 | 今回作ったジャーニーマップは、次回（009）何に使われる？ | サイトマップと導線設計のインプット（各ステップに対応するページを設計する） |
