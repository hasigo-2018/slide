# 構成指示書：016 Logic Prompt：戦略をAI指示に翻訳する技術【実践編】

## 基本情報
- **テーマ:** Logic Prompt：戦略をAI指示に翻訳する技術
- **目的:** TALENTBOARDの3ペルソナに対してLogic Promptの型を使いこなし、AIの出力品質を評価・改善できるようになる
- **推奨スライド数:** 12〜15枚
- **推奨時間:** 約50分×3コマ（150分）
- **使用ツール:** ChatGPT
- **ケーススタディ:** TALENTBOARD（人材紹介会社向けクラウド型採用管理ツール）

## 重要な設計意図
- 「設計シートを埋める→ChatGPTで回す→評価→改善」の4ステップを1サイクル回しきることが最優先
- 「自分で評価してからAIの出力と比較する」パターンを確立する（001実践編の「100文字で書く→ChatGPTと比較」と同じ思想）
- 演習3の「特定→仮説→修正→検証」サイクルは020（カスタムGPT構築）のPDCAと同じ思考パターン。ここで体に覚えさせる
- 成果物（3ペルソナ分のプロンプト＋評価済み出力）はBlock 5のLP制作でそのまま使用する
- Block 2で培ったペルソナへの解像度がプロンプトのContext品質を決める。スライド3の検証ワークで「整骨院でやった視点」をTALENTBOARDに転用する場を意図的に作る

## ビジュアルアーキテクト育成上の位置づけ

このテーマは、題材「Logic Prompt：戦略をAI指示に翻訳する技術」を通じて、ビジュアルアーキテクトに必要な「目的に沿って設計・判断・説明する力」を鍛える位置づけである。単に作業手順やツール操作を覚えるのではなく、このテーマの目的（TALENTBOARDの3ペルソナに対してLogic Promptの型を使いこなし、AIの出力品質を評価・改善できる状態）に向けて、何を選び、なぜそうするかを自分の言葉で説明できる状態をゴールとする。

## 視覚化方針

このテーマでは、文章説明だけでなく、必要に応じてSVG図解・画像生成・既存アセットを使い分ける。特に、初学者が状況を想像しづらい箇所、Before/After、判断基準、成果物イメージ、実務での使用場面は視覚化候補として扱う。スライド生成時は「画像があると理解が速くなるか」「SVGで構造化した方がよいか」「既存素材で具体化できるか」を確認し、視覚要素が学習目的を支える場合に採用する。

| 2 | **新規** | standard | **今日の実践目標。** ゴール＝3ペルソナ×各1本の「完成プロンプト」だけでなく、各ペルソナへの仮説・AIレビュー・改善記録を手元に持って帰る。実践の流れタイル：演習1（自分で仮説を立てる）→演習2（仮説をAIに壁打ちする）→演習3（差分を読んで改善する）。動機づけ：「今日の成果物は"捨てる練習"ではなく、Block 5のLP制作でそのまま使う素材です」 |
| 3 | **新規・核** | standard | **Block 2の目で3ペルソナを読む。** 3ペルソナの一覧表を提示。ただし「見るだけ」ではなく、Block 2（008）で整骨院に使った問いで検証させる。①このペルソナは平均的すぎないか？エッジはあるか？②行動の理由が見えるか？なぜこのツールを探すのか？③JTBDの状況が描けるか？「〔状況〕のとき、〔ジョブ〕したい。そうすれば〔望む結果〕になるから」の形式で1パターン書けるか。指示：最も「リアルに感じにくい」ペルソナに印をつける。そのペルソナへのプロンプト設計が最も苦しくなる。それが自分の課題。整骨院でやった視点がここで使われている。Block 5への接続：ここで作るプロンプトとその出力はLP制作で直接使用する |
| 4 | **新規** | standard | **演習1の概要と指示。** 課題＝TALENTBOARDの「一元管理」機能を3ペルソナそれぞれに刺さるコピーとして書かせる前に、まず自分で「このペルソナには何が刺さるか」の仮説を立てる。条件：理論編の3層構造を必ず使う / まず人事部長から / ChatGPTにはまだ投げない（仮説設計フェーズ）。制限時間：15分。配布物：仮説つきプロンプト設計シート |
| 5 | **新規** | standard | **仮説つきプロンプト設計シートの説明。** テンプレート構造を掲載：【前提条件】ターゲット・課題・サービス・使用場面 → 【自分の仮説】刺さる訴求軸・理由・自信がない点 → 【AIに検討してほしいこと】妥当性・反論・抜け漏れ・代替切り口 → 【Instruction】出力形式・出力数・トーン → 【Constraint】禁止事項・必須要素。このシートを埋めてからChatGPTに投げる |
| 6 | **新規** | standard | **人事部長ペルソナの設計ヒント。** Context設計：50代人事部長は「経営層に説明できる根拠」を求める。「コスト削減」「離職率」など数字で語れる課題を明記。Instruction設計：BtoBトーンで20〜30文字を推奨。Constraint設計：「革命」「画期的」等の安っぽい誇大表現は禁止。注意：これはヒントであって正解ではない |
| 7 | **新規** | standard | **演習2の概要と指示。** 課題＝仮説つきプロンプトをChatGPTに投入し、出力だけでなく仮説へのレビューも受け取る。手順：①設計シートを自然文に変換 ②ChatGPTに投入 ③AIの反論・抜け漏れ・代替切り口を確認 ④出力5案を評価基準で採点。評価基準：ペルソナ適合度★1〜5 / 具体性★1〜5 / トーン一貫性★1〜5。制限時間：20分 |
| 8 | **新規** | standard | **ChatGPTプロンプト投入のデモ。** 講師がリアルタイムでデモ。設計シート→自然文への変換例を提示。コピーボタン付きプロンプト例（prompt-chatgpt-016-1）：ロールプレイ指示＋前提条件＋自分の仮説（人事部長には経営説明・数字・リスク低減が刺さる等）＋AIに検討してほしいこと（反論・抜け漏れ・代替訴求）＋Instruction＋Constraint。このプロンプト自体が「仮説をAIにぶつける実例」 |
| 9 | **新規** | standard | **出力と差分の評価ワーク。** 評価シートの使い方を表で提示：案番号 / コピー / ペルソナ適合度 / 具体性 / トーン一貫性 / AIが追加した視点 / 自分が採用する理由。評価のコツ：「なんとなく良い」はNG、必ず3軸で採点 / AIの反論を受け入れるかどうかも自分で判断する。Block 6（論理的言語化）への伏線 |

## クイズ（5問・実践判断型）

| No. | 問題 | 正解のポイント |
|-----|------|---------------|
| Q1 | プロンプト設計シートで最初に埋めるべきものは何ですか？ | 前提条件と自分の仮説。ターゲットと課題を踏まえた仮説がないと、AI出力を評価できない |
| Q2 | AI出力の評価で「なんとなく良い」を排除するために使う3つの軸は？ | ペルソナ適合度・具体性・トーン一貫性 |
| Q3 | 同じ「一元管理」機能でも人事部長とリクルーターで出力が変わる最大の理由は？ | Contextに書いたペルソナの課題が異なるから |
| Q4 | AI出力の品質が低かったとき、最初にすべきことは？ | 3軸のどこが弱いか特定し、3層のどこが原因か仮説を立てる（いきなり書き直さない） |
| Q5 | 今日作った成果物（プロンプト・出力・改善記録）は今後どこで使いますか？ | Block 5のLP制作で直接使用する |
