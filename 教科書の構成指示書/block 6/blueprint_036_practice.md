# 構成指示書：036 【公開添削②】改善案の提示と修正指示ロールプレイ【実践編】

## 基本情報
- **テーマ:** 【公開添削②】改善案の提示と修正指示ロールプレイ
- **目的:** 受講生同士のTALENTBOARD LPを交換添削し、SBI+Iモデルで修正指示を伝え、受け取る側の体験もすることで、ディレクションの双方向フローを完成させる
- **推奨スライド数:** 16〜18枚
- **推奨時間:** 約50分×5コマ（250分）
- **使用ツール:** Figma（赤入れ・コメント）、AI Chat（ChatGPT / Gemini など）（分析補助）
- **素材:** 受講生同士のTALENTBOARD LP（Figmaファイル＋030のエビデンスv2）

## 重要な設計意図
- 4フェーズ構成：Phase 1（交換添削）→ Phase 2（修正指示ロールプレイ）→ Phase 3（役割交代で再実施）→ Phase 4（振り返り＋Block 6総括）
- 「ディレクター役」と「制作者役」の両方を全員が体験する設計。出す側と受ける側の両方を経験して初めてフィードバックの質がわかる
- 033の3原則＋SBI+Iモデルを全フェーズで徹底。035のラウンド2（他者のバナー選定→制作者に共有）の拡大版として位置づけ
- Block 6の最終テーマとして、030–035の全スキルの統合確認を行う。各フェーズで「どのスキルを使っているか」を受講生自身に自覚させる
- Block 7（チーム連携）への橋渡し。036の「他者との対話でデザインを前に進める」体験が、Block 7の協業スキルの基盤になる
- AI Chatは添削の代行ではなく、見落としチェックと表現の整えに限定する。相手に伝える最終判断と伝え方は、必ず自分で決める

## ペア編成（講師用ガイド）

| 基準 | 内容 |
|------|------|
| 編成方法 | 講師がペアを指定する（受講生の自由選択にしない）。スキルレベルが近い者同士のほうがフィードバックの質が均等になりやすい |
| 奇数の場合 | 3人グループを1つ作り、順繰りに添削を回す（A→Bを添削、B→Cを添削、C→Aを添削） |
| 注意点 | 仲が良すぎるペアは馴れ合いになりやすい。適度な距離感のペアが望ましい |

## ビジュアルアーキテクト育成上の位置づけ

## クイズ（5問・実践判断型）

| No. | 問題 | 正解のポイント |
|-----|------|---------------|
| Q1 | 036のロールプレイで「良い点を先に伝える」ことが033以上に重要な理由は？ | 制作者が目の前にいるため。外部LPの制作者は教室にいないが、仲間は目の前で聞いている。最初にいきなり改善点から入ると、制作者は防衛的になりフィードバックを受け取れなくなる。良い点から入ることで「この人は自分の作品を理解した上で指摘している」という信頼が生まれる |
| Q2 | 制作者役が「不同意」を表明するとき、必要なものは何か？ | 030のエビデンス3層構造に基づく根拠。「なんとなく嫌だ」は不同意ではなく感情的反発。「025のCTA設計の"心理的安全"の観点からこうしています」のように理論的根拠を示して初めて「根拠を持った判断」になる |
| Q3 | 「ディレクター役」と「制作者役」の両方を体験させる設計意図は？ | フィードバックの質は出す側だけで決まらない。受け取る側の姿勢（聞く→メモ→深掘る→判断）が整って初めて対話が成立する。実務ではクライアントの前でディレクター役になることも、先輩ディレクターからフィードバックを受ける制作者役になることもある。両方の立場を体験することで、どちらの立場でも建設的な対話ができる |
| Q4 | Phase 4の振り返りで「出す側と受ける側のどちらが難しかったか」を問う理由は？ | 受講生によって得意な立場が異なる。「出すのは得意だが受けるのが苦手」な人は防衛的になりやすく、「受けるのは得意だが出すのが苦手」な人は遠慮して本質を突けない。自分の傾向を自覚することが、Block 7以降のチーム連携での課題発見につながる |
| Q5 | Block 6全体（030–036）で完成した「裁く力」の全体像を説明せよ | 030：自分のデザインを言語化する力。031：他者の曖昧な言葉を翻訳する力。032：修正指示をFigmaで伝える力。033：初見の作品を論理的に分析する力。034：データで判断を裏付ける力。035：大量候補から1案を選ぶ力。036：仲間と対話しながらデザインを前に進める力。これら7つが揃って「ビジュアルアーキテクトの裁く力」になる |
